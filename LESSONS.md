# LESSONS.md — 검수 학습 원장 (compounding QA ledger)

이 파일은 **검수 에이전트**가 매 리서치 작업의 결함·수정·방지규칙을 누적하는 곳이다.
작성·만화 에이전트는 **매 작업 시작 시 아래 "방지 규칙 체크리스트"를 프리플라이트로 로드**하여 같은 류의 실수를 사전 차단한다(품질 복리).

## 방지 규칙 체크리스트 (작업 시작 시 로드)
- [ ] 출처 충돌(같은 사실에 다른 수치/명령)이 있으면 단정하지 말고 출처별로 병기한다.
- [ ] 단일 검색 요약에만 나오는 수치는 1차 출처로 교차검증되기 전엔 "한 자료에 따르면"으로 한정한다.
- [ ] 설치 명령/버전/스타수처럼 시간에 민감한 값은 조회 날짜를 함께 적는다.
- [ ] 국내·해외 출처를 모두 인용하고, 해외 인용엔 한글 요약/번역을 붙인다.
- [ ] 만화 대사는 본문 fact-ledger 밖의 새 사실을 만들지 않는다(각색만).
- [ ] **DEEP는 개요+목록이 아니다.** why/how/비교표/트레이드오프/실전 워크스루/비용·보안까지. QUICK 확장본이면 FAIL. 소스 10+개·병렬 리서치.
- [ ] **"학습만화 = 캐릭터툰" 가정 금지.** 사용자 디자인 스펙(프리미엄 인포그래픽 학습카드: 웜화이트·라운드서체·여백·모티프) 우선. 임베드는 작게(기본 1장 4분할 요약카드).
- [ ] **비주얼 톤을 멋대로 정하지 말 것.** 갤러리/카드는 사용자 톤(라이트·웜화이트) 먼저 확인. 다크 기본 가정 금지. 큰 변경 전 Figma 시안으로 확인.
- [ ] **개념 설명서 = 투트랙(이해편+확장편)(사용자 확정 2026-06-21, 최종).** 개념·학습형은 목적이 다른 두 문서로: **이해편**(slug=`<concept>`, 입문·딥, 비유·예제·"왜 자연스러운가"·대표 응용1) + **확장편**(slug=`<concept>-advanced`, 학부+대학원, 정의·유도·여러 응용·개념 발전 지도·최전선, 이해편 전제). **둘 다 DEEP, 역할 분담으로 중복 금지**, 문체 평어, 끝맺음 상호 링크. 골격·분량 테이블은 PIPELINE.md "개념 설명서=투트랙" 절. 교훈 누적: ①3계층 한 파일=얕음 ②레벨 4분할=얕고 파편화 ③단일 딥=입문 독자엔 인트로 부족 → 목적축 2분할이 최선.
- [ ] **시리즈 모드(사용자 트리거 "시리즈로 작성", 2026-06-23).** 단일 문서 강박 금지 — 담을 게 많으면 여러 파일로. "시리즈" 지시 시 **메인 1편 + 내가 선정·사전보고하는 파생 주제 3편 = 4편 DEEP**, 각 편 독립 게이트 통과, frontmatter `시리즈/시리즈순서` + 본문 끝 내비게이션(◀이전/▶다음/목차), 공통 slug 접두사, 각 편 파이프라인 완주. 상세는 PIPELINE.md "시리즈 모드" 절.
- [ ] **개념 설명서 분량·요소 객관 카운트 게이트(2026-06-22 글자수로 보정).** 커밋 전 측정: **★정본 글자수 `wc -m`(이해편 ≥15,000 / 확장편 ≥20,000)** · 줄수 `wc -l`(≥140 / ≥165) · 섹션 `^## `(≥8 / ≥10) · 시각화(≥3 / ≥2) · 출처(≥7 / ≥12). 하한 미달이면 **머지 금지·보강**, PR 본문에 측정값 표 첨부. **단어수 `wc -w`는 참고만**(한국어+수식은 30~40% 낮게 찍혀 부당 탈락 — gold standard 이란-미국 25,104자 대비 비율로 판정). 주관 판단 금지, 자문: "이란-미국만큼 읽을 게 있나?"

## 2026-06-21 · 이란-미국 분쟁 타임라인 (자가검수)

- 무엇이 관찰됐나:
  1. 클라우드 세션에서 **WebFetch가 다수 사이트(naver terms, aljazeera, npr, wikipedia, 의회도서관)에서 403** 반환 → 1차 문서 정독 실패. WebSearch 요약 + Naver MCP 스니펫으로 교차검증해 대체.
  2. 2026년 사안에서 국내 보도가 **2025년 '12일 전쟁'과 2026년 '106일 전쟁'을 혼용**하는 경우 발견 → 두 라운드를 명시적으로 분리 서술하고 본문에 '출처 충돌·주의' 박스로 한정.
- 수정: 최신·유동 구간은 (a) 조회 날짜(2026-06-21) 병기, (b) "진행 중 사안" 라벨, (c) 보도 간 미세차(서명일·조항 수)는 골자 위주로 한정, (d) 재검증 시점(MOU 60일 후) 명시.
- 다음 방지 규칙:
  - [ ] 클라우드 세션은 **WebFetch 403을 전제**로 설계 — WebSearch 요약 + Naver MCP 스니펫 다중교차로 사실 확정, 단일 스니펫 단정 금지.
  - [ ] 동일 명칭의 사건이 **연도만 다른 별개 사건**일 수 있음(예: 12일 전쟁 vs 106일 전쟁) — 연도·기간으로 구분 검증 후 서술.

## 2026-06-22 · 벡터 보고서 + 게이트 지표 보정 (첫 투트랙 적용)
- 결함유형: 측정 지표 오보정(`wc -w`가 한국어+수식 과소측정)
- 무엇이 관찰됐나:
  1. 벡터 이해편/확장편을 새 표준(투트랙+게이트)으로 작성. 첫 측정에서 단어수 게이트(900/1300)에 미달(643/709) → **머지 보류하고 실질 내용 보강 반복**(worked 예제·부분공간·차원정리·노름·힐베르트 내적 등 추가).
  2. 그래도 단어수가 더디게 차올라(이해편 877, 확장편 1064) 검증 차 gold standard와 직접 비교 → **이미 DEEP으로 인정된 hermes-deep가 742단어**로 벡터 확장편(1064)보다 짧음을 발견. 글자수로 보면 확장편 22,191 vs 이란-미국 25,104(88%)로 동급.
  3. 결론: `wc -w`가 한국어 조사·수식 줄 때문에 수학 DEEP을 부당 탈락시킴. **내용은 충분, 지표가 틀림.**
- 수정: 게이트 정본 지표를 **글자수 `wc -m`·줄수**로 전환(이해편 ≥15,000자/140줄, 확장편 ≥20,000자/165줄), 단어수는 참고로 강등. 근거 수치를 PIPELINE에 명기(게이트를 몰래 낮춘 게 아니라 지표 보정임을 투명화). 보강 후 벡터 이해편 18,191자·확장편 22,191자로 **정본 게이트 통과**.
- 다음 방지 규칙:
  - [ ] 한국어·수식 문서의 분량은 **글자수(`wc -m`)로 측정**한다(단어수 맹신 금지).
  - [ ] 지표가 내용과 어긋나면 **gold standard와 직접 비교**해 보정하고 근거를 남긴다 — 단, 보정과 '게이트 회피'를 구분(비교 수치 공개).
