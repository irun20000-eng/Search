# LESSONS.md — 검수 학습 원장 (compounding QA ledger)

이 파일은 **검수 에이전트**가 매 리서치 작업의 결함·수정·방지규칙을 누적하는 곳이다.
작성·만화 에이전트는 **매 작업 시작 시 아래 "방지 규칙 체크리스트"를 프리플라이트로 로드**하여 같은 류의 실수를 사전 차단한다(품질 복리).

## 방지 규칙 체크리스트 (작업 시작 시 로드)
- [ ] 출처 충돌(같은 사실에 다른 수치/명령)이 있으면 단정하지 말고 출처별로 병기한다.
- [ ] 단일 검색 요약에만 나오는 수치는 1차 출처로 교차검증되기 전엔 "한 자료에 따르면"으로 한정한다.
- [ ] 설치 명령/버전/스타수처럼 시간에 민감한 값은 조회 날짜를 함께 적는다.
- [ ] 국내·해외 출처를 모두 인용하고, 해외 인용엔 한글 요약/번역을 붙인다.
- [ ] 만화 대사는 본문 fact-ledger 밖의 새 사실을 만들지 않는다(각색만).
- [ ] **DEEP는 개요+목록이 아니다.** why/how/비교표/트레이드오프/실전 워크스루/비용·보안까지. QUICK 확장본이면 FAIL. 소스 10+개·병렬 리서치.
- [ ] **"학습만화 = 캐릭터툰" 가정 금지.** 사용자 디자인 스펙(프리미엄 인포그래픽 학습카드: 웜화이트·라운드서체·여백·모티프) 우선. 임베드는 작게(기본 1장 4분할 요약카드).
- [ ] **비주얼 톤을 멋대로 정하지 말 것.** 갤러리/카드는 사용자 톤(라이트·웜화이트) 먼저 확인. 다크 기본 가정 금지. 큰 변경 전 Figma 시안으로 확인.

## 2026-06-21 · 헤르메스 에이전트 (사용자 피드백)
- 결함유형: 깊이부족 / 스타일이탈(방향오인) / 비주얼톤 오인
- 무엇이 틀렸나:
  1. DEEP가 전혀 deep하지 않고 QUICK 확장 수준(개요+설치+목록). why/how/비교/실전이 없음.
  2. 코믹을 캐릭터툰(로봇·사람 말풍선)으로 만듦. 사용자가 원한 건 "프리미엄 인포그래픽 학습카드"(웜화이트·라운드서체·여백). 품질 "최악, 없는 게 나을 정도".
  3. 임베드 컷이 920px로 과대. "한 컷에 네 내용 담을 사이즈면 족함".
  4. 갤러리를 다크로 만듦. 사용자는 밝은 톤 선호.
- 수정: DEEP 풀 심층 재작성(병렬 리서치·소스10+·비교표·워크스루·Slack·비용/보안). 코믹=클린 학습카드(auto) 1장 4분할로 축소. 갤러리=라이트/웜화이트 재설계. 모두 Figma 시안 먼저 확인 후 코드.
- 다음 방지 규칙: 위 체크리스트의 마지막 3줄(DEEP 기준·캐릭터툰 금지·비주얼톤 사전확인) 상시 적용. 산출 전 "이게 정말 DEEP/사용자 톤인가" 자문.
