# 벤포드 법칙(Benford's Law) — 확장편

> 이해편 [[벤포드 법칙]] 이 "왜 1이 많은가"를 그림으로 납득시켰다면, 확장편은 **로그 식의 유도, 스케일·진수 불변성, 일반 유효숫자 법칙, 적합도 검정으로 조작을 판정하는 법**, 그리고 포렌식·이미지·AI 탐지로 뻗는 발전 지도를 다룬다. 이해편의 직관을 전제한다.

## TL;DR
- 벤포드 법칙의 본질은 **"유효숫자(significand) S의 상용로그 log₁₀S가 [0,1)에서 균등분포"**라는 한 문장으로 압축된다. 여기서 **P(첫자리=d)=log₁₀(1+1/d)**가 즉시 따라 나온다 [6][7].
- 이 균등성은 **스케일 불변성**(단위 배율에 불변)과 **진수 불변성**(base에 무관)에서 강제된다 — 그 둘을 만족하는 유효숫자 분포는 **로그분포가 유일**하다 [8][10].
- 더 근본적 설명은 **Hill의 정리(1995)**: "여러 분포에서 무작위로 뽑아 섞은 표본"의 유효숫자는 벤포드로 수렴한다. 곱셈형 과정(로그가 합 → CLT)이 대표적 생성기다 [6][7].
- 실무 판정은 **적합도 검정**: 카이제곱(χ²), 평균절대편차(MAD), 디짓별 Z-검정. 포렌식 회계의 표준 절차다(Nigrini) [11][12(국내)].
- 응용 최전선: **포렌식 회계·선거 포렌식·과학 데이터 위조·JPEG 이미지 위조·블록체인·LLM 생성문 탐지**. 단, **남용·오판 논란**(선거 판정의 타당성 등)도 크다 [6][9].

## 핵심 포인트
1. **첫자리 사건 = 로그 유효숫자의 구간 사건.** 이 재해석이 모든 유도의 열쇠다 [6].
2. **로그 균등 → 벤포드**는 계산 한 줄. 문제는 "왜 로그가 균등하냐"이고, 답이 **불변성 + 혼합정리**다 [8].
3. **일반 유효숫자 법칙**: 앞 k자리 D에 대해 P=log₁₀(1+1/D). 둘째 자리부터는 급속히 평평해진다 [6].
4. **검정은 표본크기에 민감.** n이 크면 χ²가 사소한 편차도 기각한다 → MAD·효과크기 병행 [11].
5. **벤포드는 필요조건이지 충분조건이 아니다.** 위반=의심 신호, 부합=무죄 아님 [12(국내)].

## 1. 직관 압축 (이해편 요약)

값 x가 **비율(곱셈)로 자라며 여러 자릿수에 퍼지면**, 로그축에서 고르게 흩어지고, 로그축의 "1~2 칸"이 "8~9 칸"보다 넓어 **첫자리 1이 30.1%**가 된다. 스케일을 바꿔도(×상수) 로그축에선 **평행이동**일 뿐이라 분포가 불변이다. 이 한 단락이 이하 전개의 씨앗이다.

확장편의 서술 전략은 이 직관을 **세 층위**로 정밀화하는 것이다. **(1) 대수적 층위** — 첫자리 사건을 유효숫자 로그의 구간 사건으로 옮겨 벤포드 식을 계산으로 유도(§2). **(2) 구조적 층위** — 그 로그가 왜 균등한지를 불변성·혼합·측도로 설명(§3). **(3) 통계적 층위** — 실데이터가 이 이상(理想)에서 얼마나 벗어났는지를 검정으로 판정(§4). 이해편이 "그림"이었다면 확장편은 이 세 층위를 잇는 "다리"다. 각 층위는 독립적으로도 쓸모가 있다: 응용가는 (3)만, 이론가는 (2)만 필요할 수 있다.

## 2. 정의 · 메커니즘 — 유효숫자와 맨티사

양수 x를 과학적 표기로 쓴다:
```
x = S(x) · 10^k,   S(x) ∈ [1, 10),  k = ⌊log₁₀ x⌋ ∈ ℤ
```
여기서 **S(x)를 유효숫자(significand)**라 한다. 첫자리 사건을 로그로 옮기면:
```
첫자리(x) = d  ⟺  S(x) ∈ [d, d+1)
            ⟺  log₁₀ S(x) ∈ [log₁₀ d, log₁₀(d+1))
```
이제 **가정**을 놓는다: 확률변수 Y = log₁₀ S(x) 가 **[0,1)에서 균등분포**. 그러면
```
P(첫자리 = d) = P(Y ∈ [log₁₀ d, log₁₀(d+1)))
             = log₁₀(d+1) − log₁₀ d
             = log₁₀( (d+1)/d )
             = log₁₀(1 + 1/d)
```
이것이 벤포드 식이다 [6][7]. **모든 것이 "Y가 균등이다"라는 가정에 달려 있다.** §3이 그 가정의 정당화다.

**측도론적 정식화(요약).** 엄밀히는 유효숫자 함수 S: ℝ⁺→[1,10)에 대해 "유효숫자 σ-대수" 𝒮를 정의하고, 벤포드 확률측도 P_B를 P_B(S ∈ [1,t)) = log₁₀ t (1≤t<10)로 둔다. 이때 첫자리 사건 {S∈[d,d+1)}의 측도가 log₁₀(1+1/d)로 나온다. 이 틀에서 "스케일 불변"은 곱셈군 (ℝ⁺,×)의 작용에 대한 **불변측도(하르 측도)**가 로그측도임과 동치다. 즉 벤포드 분포는 **곱셈 구조가 자연히 유도하는 측도**다 — 덧셈 구조에서 르베그측도가 자연스러운 것과 평행하다 [10]. 이 관점은 "왜 벤포드가 여기저기 튀어나오나"를 **군론적 필연**으로 재서술한다.

## 3. 왜 로그가 균등한가 — 세 갈래 유도

### (a) 스케일 불변성으로부터
데이터에 상수 c>0를 곱하면 log₁₀(cx)=log₁₀c + log₁₀x. 유효숫자의 로그 Y는 **Y + log₁₀c (mod 1)**로 원환(circle) 위에서 **회전**한다. "임의의 회전에 대해 불변인 [0,1) 위의 분포"는 **균등분포가 유일**하다(원 위 하르 측도). 따라서 스케일 불변을 요구하면 Y는 균등, 즉 벤포드가 강제된다 [8][10].

조금 더 형식적으로 보자. Y의 분포함수를 F, 밀도를 f라 하면, 회전 불변성은 모든 θ에 대해 f(y)=f(y+θ mod 1)를 요구한다. y를 고정하고 θ를 훑으면 f가 상수여야 함이 즉시 나오고, ∫₀¹ f =1 이므로 f≡1, 곧 균등이다. 이 논증의 힘은 **"단 하나의 요건(단위 무관)만으로 분포 형태가 완전히 결정된다"**는 데 있다. 물리 법칙이 단위계에 무관해야 하듯, 자연 수량의 유효숫자 분포가 단위에 무관하려면 벤포드일 수밖에 없다. 다만 실데이터가 **엄밀히** 스케일 불변인 경우는 드물며(유한 범위·절단 존재), 그래서 (c)의 혼합 논증이 "왜 근사적으로 흔한가"를 보완한다 [8].

### (b) 진수(base) 불변성
같은 논리를 밑 b에 대해 펴면, **모든 밑에서 동시에 성립하는 유효숫자 분포도 로그분포뿐**이다. 벤포드가 "특정 진법의 우연"이 아니라 구조적임을 보여준다 [10].

### (c) 혼합·곱셈형 생성 (Hill 정리, 곱셈 CLT)
현실 데이터가 정확히 스케일 불변인 경우는 드물다. 더 현실적인 설명 둘:
- **곱셈형 중심극한**: X = ∏Xᵢ 이면 log X = Σ log Xᵢ 가 정규로 수렴(CLT). 분산이 커지면(여러 자릿수) 정규분포를 1주기로 접은 Y가 **균등에 근접** → 벤포드.
- **Hill(1995) 정리**: "여러 확률분포를 무작위로 고르고 거기서 표본을 뽑아 섞으면", 그 결합표본의 유효숫자 분포는 **벤포드로 수렴**한다. "자연이 여러 과정을 섞는다"는 관찰과 맞물린다 [6][7].

세 갈래는 서로 보완적이다: 불변성은 **유일성**을, 혼합정리는 **왜 흔한지**를 설명한다.

**곱셈 CLT를 조금 더.** log X = Σ_{i=1}^{n} log Xᵢ 가 평균 μ·표준편차 σ의 정규 N(μ, σ²)로 근사된다고 하자(로그정규, lognormal). 유효숫자의 로그 Y = log X mod 1 의 분포는, N(μ,σ²)를 폭 1로 접어 겹친 것(wrapped normal)이다. σ가 커질수록(데이터가 여러 자릿수에 걸칠수록) 이 접힘이 촘촘해져 **[0,1)에서 균등에 지수적으로 빠르게 수렴**한다(푸리에 계수가 e^(−2π²σ²k²)로 감쇠). 즉 **"σ가 1보다 충분히 크면(대략 여러 桁) 벤포드가 매우 잘 맞는다"**는 정량적 기준이 나온다. 반대로 σ가 작으면(좁은 범위) 벤포드에서 벗어난다 — 이해편의 "여러 자릿수 조건"이 여기서 수식으로 설명된다 [7].

## 4. 대표 응용 (Worked)

### Worked 1 — 손으로 확률 계산·둘째 자리
첫자리: P(1)=log₁₀2=0.3010, P(9)=log₁₀(10/9)=0.0458. 합은 Σ_{d=1}^{9} log₁₀(1+1/d)=log₁₀10=1 (완비).

**둘째 자리** 법칙: 둘째 자리가 d일 확률은 앞자리를 모두 더해
```
P(둘째자리 = d) = Σ_{k=1}^{9} log₁₀(1 + 1/(10k + d))
```
계산하면 0은 약 12.0%, 9는 약 8.5%로 **첫자리보다 훨씬 평평**하다. 셋째 자리는 거의 균등(10%)에 수렴한다 [6]. **일반형**: 앞 k자리 정수 D에 대해 P=log₁₀(1+1/D).

### Worked 2 — 카이제곱 적합도 검정으로 조작 판정
어느 회사 장부에서 **첫자리 도수**를 관측했다고 하자(n=1000).

| d | 관측 Oₐ | 기대비율 | 기대 Eₐ=1000·log₁₀(1+1/d) |
|---|---|---|---|
| 1 | 250 | 0.301 | 301 |
| 2 | 200 | 0.176 | 176 |
| 3 | 150 | 0.125 | 125 |
| 4~9 | 400 | 0.398 | 398 |

검정통계량:
```
χ² = Σ (Oₐ − Eₐ)² / Eₐ
   ≈ (250−301)²/301 + (200−176)²/176 + (150−125)²/125 + …
   ≈ 8.64 + 3.27 + 5.00 + … 
자유도 df = 9−1 = 8, 유의수준 0.05의 임계값 ≈ 15.51
```
이 값이 임계값을 넘으면 "벤포드에서 유의하게 벗어남"으로 **조작 의심 플래그**를 세운다. 실무에선 표본이 매우 크면 χ²가 과민해지므로 **평균절대편차(MAD)**나 디짓별 Z를 병행한다 [11][12(국내)]. 포렌식 회계의 표준(Nigrini)이 이 절차다.

### Worked 3 — 스케일 불변성 수치로 확인
간단한 벤포드 표본에 상수를 곱해도 첫자리 분포가 유지됨을 손으로 볼 수 있다. **2의 거듭제곱** 1,2,4,…,2⁶⁰(첫자리 분포가 벤포드에 근접)에 **각각 3을 곱한** 3,6,12,24,48,96,192,… 를 만들어 첫자리를 세보자. 3·2ᵏ의 첫자리 열은 3,6,1,2,4,9,1,3,… 로, 다시 **1이 최빈**이 되고 전체 분포는 원본과 통계적으로 구별되지 않는다. c=3 대신 c=7, c=0.5 무엇을 곱해도 결과는 같다 — 로그축에서 **평행이동(mod 1)**일 뿐이기 때문이다. (엑셀로 `=3*2^행`과 `=LEFT(TEXT(값,"0"),1)`을 60행 만들어 확인 가능.) 이 실험은 §3(a)의 회전 불변성을 눈으로 보여준다 [8].

### Worked 4 — 포렌식 회계의 검정군(Nigrini suite)
실무에서는 첫자리 하나만 보지 않고 **여러 벤포드 검정을 묶어** 쓴다 [11][12(국내)][4(국내)].
```
① First-Digit Test        : 첫자리 1~9 분포 (거시 스크리닝)
② First-Two-Digits Test   : 앞 두 자리 10~99 분포 (10~99, 정밀 — 특정 값 급증 포착)
③ Last-Two-Digits Test    : 끝 두 자리 (반올림·조작·중복 탐지, 균등이 기대)
④ Number Duplication Test : 특정 금액의 비정상 반복 (결재 한도 회피)
⑤ Second-Order Test       : 정렬 후 인접차의 분포 (데이터 무결성)
```
예컨대 **결재 한도 500만 원**을 피하려 480만~499만 원을 반복 청구하면, **First-Two-Digits Test에서 48·49가 급증**하는 스파이크로 드러난다. 이처럼 검정은 "어떤 자리를 보느냐"에 따라 서로 다른 조작 유형을 잡는다. 각 검정은 §4의 χ²/MAD를 그 자리 분포에 적용한 것이다.

### 4.5 응용 최전선 — 회계 너머로

벤포드는 "유효숫자가 로그균등"이라는 성질만 있으면 도메인을 가리지 않는다. 그래서 회계를 훌쩍 넘어 번진다 [6][9][5(국내)].

- **이미지 포렌식**: 자연 이미지를 JPEG 압축하면 **DCT(이산코사인변환) 계수**가 벤포드에 가까운 분포를 갖는다. 이중 압축·변조가 일어나면 이 분포가 **깨지므로**, 위조 이미지·딥페이크 탐지의 특징으로 쓴다. "픽셀이 아니라 변환계수의 첫자리"를 보는 것이 핵심이다.
- **과학 데이터 무결성**: 논문에 실린 회귀계수·측정값의 첫자리가 벤포드에서 크게 벗어나면 **데이터 날조(fabrication)** 의심 신호가 된다. 실제로 일부 연구부정 사건에서 보조 증거로 쓰였다.
- **물리·천문 스펙트럼**: 원자 스펙트럼선 세기, 물리상수 표(벤포드 원 데이터에 이미 포함)도 벤포드를 보인다. 데이터베이스 오류 스크리닝에 활용된다 [6].
- **거시·경제 데이터**: 국가가 보고하는 GDP·재정 통계의 벤포드 이탈이 **통계 조작** 의심에 쓰인 사례가 있다(그리스 재정통계 논쟁 등).
- **블록체인·네트워크**: 트랜잭션 금액, 소셜 네트워크의 친구 수·좋아요 수도 벤포드를 따르는 경향이 보고됐다 — 봇·어뷰징 탐지 특징.
- **LLM/AI 생성문 탐지**: 사람이 쓴 글과 모델이 생성한 글의 **숫자·통계적 패턴 차이**를 재는 특징으로 벤포드류 지표가 실험된다. 국내에서도 "창작자 언어 분포"와 연결해 논의된다 [5(국내)].

공통 교훈: **"자연스러움의 서명"**은 위조가 흉내 내기 어렵다. 다만 각 도메인마다 **적용 요건과 오판율이 다르므로**, 벤포드 이탈을 곧바로 유죄로 읽는 것은 금물이다(§5). 실무에서 벤포드가 사랑받는 이유는 **비용이 거의 0**이라는 점이다 — 데이터의 첫자리만 세면 되니 별도 라벨·모델 학습이 필요 없다. 그래서 대규모 데이터의 **1차 필터**로는 최고의 가성비를 가지되, 걸러낸 뒤엔 반드시 도메인 지식과 정밀 조사가 뒤따라야 한다.

## 5. 통합 의미 — 필요조건·경계·남용

- **필요조건이지 충분조건 아님**: 자연 데이터라고 반드시 벤포드는 아니고(범위 제약·부여번호), 벤포드를 따른다고 조작이 없는 것도 아니다. **위반은 '조사 시작', 부합은 '무죄 아님'** [12(국내)].
- **적용 요건**: (i) 여러 자릿수(3+ orders)에 걸침, (ii) 상·하한 인위 절단 없음, (iii) 부여·순번이 아님, (iv) 표본 충분. 요건 위반 시 판정이 무의미하다 [9].
- **검정력의 함정**: n이 크면 통계적 유의성이 실질적 중요성을 과장한다. **효과크기(MAD 구간 기준)**로 보정 [11].
- **남용 논란**: 선거 포렌식에서 벤포드 위반을 곧바로 부정선거로 단정하는 것은 방법론적으로 비판받는다(정상 투표도 위반할 수 있음). **보조 증거**로만 써야 한다 [6][9].

특히 **선거 데이터의 첫자리**는 이론적으로도 벤포드를 따를 이유가 약하다. 투표구별 득표수는 "여러 자릿수에 곱셈적으로 퍼진" 데이터가 아니라, **투표구 크기가 비슷하게 설계**되어 좁은 범위에 몰리는 경우가 많기 때문이다(§3 σ 조건 위반). 그래서 학계는 첫자리 대신 **둘째 자리 벤포드(2BL)**를 쓰거나, 애초에 벤포드가 부적합하다고 본다. 2009년 이란 대선 분석이 유명해졌지만, 동시에 "벤포드만으로 부정을 입증할 수 없다"는 반박도 많았다 — **도구의 적용 요건을 무시한 채 결론을 서두르면 오히려 신뢰를 잃는다**는 교훈이다 [6][9]. 이는 벤포드에 국한된 이야기가 아니라, **모든 통계적 이상탐지가 '신호'이지 '증명'이 아니라는** 일반 원칙의 한 사례다.

## 6. 🗺️ 개념 발전 지도

벤포드는 "재미있는 잡학"으로 끝나지 않고 네 방향으로 뻗는다. 아래 표는 각 방향에서 **무엇을 풀면 무엇이 열리는지**를 정리한 것이다 — 학습 로드맵으로 삼으면 된다.

| 방향 | 가정을 풀면 | 무엇이 열리나 | 어디서 쓰나 | 더 알아야 할 것 |
|---|---|---|---|---|
| **일반화** | 첫자리→앞 k자리·연속 유효숫자 | 일반 유효숫자 법칙 P=log(1+1/D), 유효숫자의 확률측도 | 정밀 포렌식·다자리 검정 | 유효숫자 σ-대수·불변측도 [10] |
| **생성이론** | 단일 분포→혼합·곱셈형 | Hill 정리, 곱셈 CLT, 스케일/진수 불변 | "어떤 데이터가 따를까" 사전판단 | 측도론·하르 측도 [6][8] |
| **검정·통계** | 눈대중→형식 검정 | χ²·MAD·Z, 베이즈 사후, 검정력 분석 | 회계·세무·과학데이터 감사 | 다중검정·효과크기 [11] |
| **응용확장** | 회계→신호·이미지·언어 | JPEG DCT계수·물리스펙트럼·LLM 생성문·블록체인 | 이미지 위조·과학사기·AI탐지 | 도메인별 위반요건·오판율 [9] |

## 7. 함정 · 체크리스트

- [ ] **적용 요건 확인 먼저**: 자릿수 범위·절단·부여번호 여부. 요건 미달이면 검정 자체가 무의미 [9].
- [ ] **표본크기 보정**: 대표본은 χ² 과민 → MAD·효과크기 병행 [11].
- [ ] **위반 ≠ 유죄**: 1차 스크리닝임을 명시. 후속 정밀조사로 연결 [12(국내)].
- [ ] **둘째 자리·요약통계**도 함께 보라(첫자리만으로 부족할 때) [6].
- [ ] **선거·정치 데이터 단정 금지**: 방법론 한계 인지, 보조 증거화 [6][9].
- [ ] **진수·단위 불변**을 이용해 데이터 전처리(단위 통일)에 흔들리지 말 것 [8].
