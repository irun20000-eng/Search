# 클로드 코워크(Claude Cowork) 완전 가이드 — 모바일 활용 중심

> 코워크는 "Claude Code의 에이전트 능력을 **코딩 밖 지식노동**으로" 확장한 기능이다. 원래 데스크톱 전용이었지만 **웹·모바일로 확장(베타, 2026-07-07 Max 우선 롤아웃)** 되면서, 이제 **폰에서 업무를 맡기고 자리를 뜨는** 사용이 가능해졌다. 이 글은 코워크가 무엇이고 → 어떤 구조인지 → **모바일에서 어떻게 쓰는지** → 예약 작업·요금·한계까지 한 편에 정리한 실무 가이드다. (조회일 2026-07-18 기준, 베타 롤아웃 중이라 화면·플랜은 변동 가능.)

## TL;DR
- **코워크는 "채팅"이 아니라 "실행"이다.** 대화창에서 **Cowork를 켜면** Claude가 실제로 **파일·폴더·앱·커넥터를 다뤄 결과물을 만들어** 낸다. 일반 대화로 돌아가려면 **Chat** 선택 [6][국내 9ansapres].
- **표면 진화: 데스크톱 → 웹·모바일(베타).** 처음엔 macOS/Windows 데스크톱 앱 전용이었으나, **2026-07-07 Max 플랜부터 웹·모바일로 롤아웃** 시작해 몇 주에 걸쳐 확대 중 [7][국내 imh1050].
- **핵심 구조 = 플러그인.** 플러그인은 **스킬 + 커넥터 + 서브에이전트**를 한 묶음으로 패키징해 역할·팀·회사에 맞춘다. 마켓플레이스로 배포, 커넥터로 Notion·Slack·Gmail·Stripe 등 연결 [국내 thkim66][국내 hfeel].
- **실행 방식 두 갈래**: ①**Dispatch(로컬)** = 내 데스크톱에서 실행 → **컴퓨터 켜두고 Claude Desktop 열려 있어야** 함. ②**클라우드/원격 세션(베타, Pro·Max)** = Anthropic 서버에서 실행 → **컴퓨터 꺼도 계속** [1][6].
- **모바일의 핵심**: **어디서든 작업 배정**. 외출 중 폰으로 "파일 찾아 요약해줘"를 던지면, Claude가 **적절한 데스크톱 세션으로 라우팅**하거나 클라우드에서 돌려 **완성물을 같은 대화로** 돌려준다. 노트북 닫아도 백그라운드로 진행 [국내 digital_snj][국내 rascalhoon].
- **예약 반복 작업**은 **모든 유료 플랜(Pro·Max·Team·Enterprise)**. 원격 실행이라 컴퓨터가 잠자도 정해진 시간에 돈다 [8].

## 핵심 포인트
1. **Cowork ↔ Chat 토글 하나.** 대화창 왼쪽 아래에서 전환 [6].
2. **모바일 = 원격 위임 창구.** 시작은 데스크톱, 확인·지시는 폰 [국내 digital_snj].
3. **플러그인(스킬·커넥터·서브에이전트)** 이 코워크를 업무 도구로 만든다 [국내 thkim66].
4. **Dispatch(로컬) vs 클라우드 세션(원격)** 구분이 모바일 사용의 열쇠 [1].
5. **예약 작업으로 반복 업무 자동화** — 유료 전체 [8].

## 1. 코워크란 무엇인가 — 채팅에서 '실행'으로

일반 클로드(claude.ai)는 **대화**한다. 글쓰기·번역·요약·아이디어 정리 같은 걸 **말로** 도와준다. **코워크(Cowork)** 는 여기서 한 발 더 나간다 — Claude가 **실제로 파일·폴더·앱·업무 도구를 조작해** 결과물을 **완성**한다. 코딩용 에이전트인 **Claude Code의 능력을, 코딩이 아닌 일반 지식노동으로 확장**한 것이 코워크다 [국내 9ansapres][6].

사용 감각의 차이가 핵심이다:
- **Chat**: "이 보고서 어떻게 쓸까?" → 조언·초안을 **텍스트로** 준다.
- **Cowork**: "이 폴더 자료로 보고서를 만들어 파일로 저장해줘" → Claude가 **폴더를 열고, 자료를 읽고, 문서를 만들어 저장**한다.

즉 코워크는 **"결과(outcome)를 설명하면, 자리를 떠도, 돌아오면 완성물"** 이라는 위임형 워크플로우다. 포맷된 문서, 정리된 파일, 종합된 리서치를 **통째로 회수**한다 [7][국내 jjw408].

## 2. 표면의 진화 — 데스크톱에서 웹·모바일로

코워크의 사용 범위는 빠르게 넓어졌다 [국내 imh1050][국내 namaksin1205][7]:
- **1단계(초기)**: **macOS·Windows 데스크톱 앱 전용**. 웹·모바일 불가. 로컬 파일을 직접 다루는 게 강점이자 제약이었다(내 컴퓨터에서만).
- **2단계(현재, 베타)**: **웹·모바일로 확장**. **2026-07-07 Max 플랜부터 롤아웃** 시작해 이후 몇 주에 걸쳐 다른 플랜으로 확대 중.

그래서 지금 시점(조회일 2026-07-18)에는 **모바일에 Cowork 옵션이 보이는 사람과 아직 안 보이는 사람**이 섞여 있다. **최신 버전으로 업데이트**했는데도 안 보이면 아직 내 계정·플랜에 롤아웃이 안 닿은 것이다 [6][국내 imh1050].

## 3. 핵심 구조 — 플러그인(스킬·커넥터·서브에이전트)

코워크를 "업무 도구"로 만드는 것은 **플러그인**이다. 플러그인은 세 가지를 **한 묶음**으로 패키징한다 [국내 thkim66][국내 jjw408]:

| 구성 요소 | 역할 |
|---|---|
| **스킬(Skill)** | 특정 작업의 절차·노하우(예: 특정 양식 문서 작성, 데이터 정리 방식) |
| **커넥터(Connector)** | 외부 앱 연결 — Notion·Slack·Gmail·Stripe 등 |
| **서브에이전트(Sub-agent)** | 복잡한 다단계·병렬 작업을 나눠 처리하는 보조 일꾼 |

이 셋을 하나로 묶어 **역할·팀·회사에 맞게 Claude를 커스터마이즈**한다. 플러그인은 **마켓플레이스**로 배포·설치하며, 예약 작업도 **설치된 플러그인·연결된 도구·스킬을 똑같이** 쓸 수 있다 [국내 hfeel][8]. 커넥터 덕분에 코워크는 "내 파일" 뿐 아니라 **Notion 문서, Slack 메시지, Gmail, 결제(Stripe)** 같은 실제 업무 데이터를 다룬다 [국내 hfeel].

> 이 "스킬 + 커넥터 + 서브에이전트" 3종 세트는 MCP·플러그인 생태계와 같은 계보다. 도구를 붙이는 표준이 궁금하면 [[클로드 MCP 가이드]] 참고.

## 4. 실행 방식 두 갈래 — Dispatch(로컬) vs 클라우드 세션(원격)

**모바일 활용을 이해하려면 이 구분이 가장 중요하다** [1][6]:

| 구분 | Dispatch(로컬 실행) | 클라우드/원격 세션 |
|---|---|---|
| 어디서 도나 | **내 데스크톱** | **Anthropic 서버** |
| 전제 조건 | 컴퓨터 **켜져 있고** Claude Desktop **열려 있어야** | 컴퓨터 **꺼져도** 계속 |
| 로컬 파일 | 데스크톱의 로컬 파일·커넥터 직접 접근 | 계정에 붙은 파일·커넥터 |
| 플랜 | — | **베타, Pro·Max** |
| 모바일 궁합 | 폰에서 지시 → **데스크톱 세션이 처리** | 폰에서 지시 → **서버가 처리, 기기 따라다님** |

- **Dispatch(로컬)** 는 "내 컴퓨터의 로컬 파일을 직접 만지는" 방식이라 강력하지만, **데스크톱을 켜둬야** 한다. 폰에서 지시해도 실제 일은 데스크톱 세션이 한다.
- **클라우드 세션** 은 서버에서 돌아 **컴퓨터를 꺼도 계속**되고, **세션·파일이 계정에 붙어 기기를 따라다닌다**. 진짜 "노트북 닫고 폰으로 확인"이 가능한 건 이쪽이다 [1].

정리: **로컬 파일을 꼭 다뤄야 하면 Dispatch(+데스크톱 켜두기)**, **이동 중 완전 위임이 목적이면 클라우드 세션**.

## 5. 📱 모바일에서 코워크 쓰기 (핵심)

### (a) 켜는 법
1. iOS/Android **클로드 앱을 최신 버전으로** 업데이트.
2. **메시지 입력창 왼쪽 아래에서 "Cowork" 선택**(안 보이면 롤아웃 미도달 → §2).
3. **원하는 결과를 설명**한다. 일반 대화로 돌아가려면 **"Chat"** [6][국내 9ansapres].

### (b) 어디서든 작업 배정(Assign from anywhere)
모바일 앱과 데스크톱 앱이 연결되면, **외출 중에도 폰으로** "파일 찾아줘 / 문서 요약해줘 / 메모 작성해줘"를 지시할 수 있다. Claude는 그 작업을 **적절한 데스크톱 세션으로 라우팅**해 처리하고, **완성물을 같은 대화로** 돌려준다. **사용자가 컴퓨터 앞에 앉아 있을 필요가 없다** [국내 digital_snj][국내 rascalhoon].

- **Dispatch(로컬)** 를 쓰는 경우: 폰에서 메시지를 보내면 **같은 대화에 결과물이 전달**된다(데스크톱이 처리) [국내 rascalhoon].
- **클라우드 세션**: 서버가 처리하므로 데스크톱이 꺼져 있어도 진행, 결과가 폰으로 온다 [1].

### (c) "맡기고 자리 뜨기"
모바일의 진짜 값어치는 **비동기 위임**이다 [7]:
- **책상에서 시작 → 폰으로 진행 확인**. 노트북을 닫아도 (클라우드 세션이면) 백그라운드로 계속.
- 긴 작업을 걸어두고 **이동·회의 중 폰으로 상태만** 본다.
- 결과가 나오면 폰에서 바로 확인·추가 지시.

## 6. 예약 반복 작업(Scheduled Tasks)

정해진 주기로 **자동 실행**하는 기능이다 [8][국내 hfeel]:
- **대상 플랜**: **모든 유료 플랜(Pro·Max·Team·Enterprise)**.
- **원격 실행**: 컴퓨터가 **잠자거나 Claude Desktop이 닫혀 있어도** 정해진 주기에 돈다.
- **각 예약 작업 = 독립 코워크 세션.** 일반 작업과 **동일한 능력**(연결된 도구·스킬·설치된 플러그인)을 쓴다.

활용 예: **매일 아침 리서치 요약**, 주간 리포트 자동 생성, 정기 파일 정리, 특정 채널 모니터링 요약 등. 모바일과 궁합이 좋다 — 예약해두고 **폰으로 결과만 회수**하면 되니까.

## 7. 글로벌·폴더별 지시와 권한

코워크는 지시를 두 층위로 관리한다 [국내 hfeel][국내 6612ty]:
- **글로벌 지시**: 모든 작업에 공통 적용되는 규칙(말투·형식·금지사항 등).
- **폴더별 지시**: 특정 폴더에서만 적용되는 규칙.

또한 **폴더 접근 권한**을 부여하면 Claude가 **그 폴더 안에서만** 작업을 수행한다. 로컬 파일을 직접 다루는 만큼 **권한 범위를 좁혀 주는 것**이 안전의 핵심이다(§12) [국내 6612ty].

## 8. 실전 워크플로우 (Worked) — 모바일 중심

### Worked 1 — 출근길 위임
지하철에서 폰으로: *"어제 회의 폴더의 녹취록을 요약해서 액션 아이템만 뽑아 문서로 저장해줘."* → Claude가 데스크톱 세션(또는 클라우드)에서 파일을 읽고 요약본을 만든다. **사무실 도착 시 완성** [국내 digital_snj].

### Worked 2 — 아침 리포트 예약 자동화
**예약 작업**으로 *"매일 오전 8시, 지정 커넥터의 어제 데이터로 일일 요약 리포트를 만들어 저장"* 을 걸어둔다. 컴퓨터를 꺼둬도 서버에서 실행 → **폰으로 결과만 확인** [8].

### Worked 3 — 원격 폴더 정리
폰에서 *"다운로드 폴더를 유형별로 분류하고 중복 파일을 정리해줘"* → 권한을 준 폴더 범위 안에서 Claude가 파일을 정리하고, **결과 요약을 같은 대화로** 보고 [국내 rascalhoon].

세 예시의 공통 골격: **폰으로 결과를 설명 → 원격/데스크톱이 실행 → 완성물 회수.** "작성"이 아니라 "지휘"다.

### 모바일에서 잘 쓰는 팁
- **지시는 '결과' 단위로 크게.** 모바일에선 긴 타이핑이 부담이니, 단계별 마이크로 지시보다 **"무엇을 만들어 어디에 저장"** 같은 결과 중심 한 문장이 효율적이다. 세부 판단은 에이전트에 맡긴다 [7].
- **음성 입력 활용.** 이동 중엔 폰 음성 받아쓰기로 지시를 부르면 빠르다.
- **긴 작업은 클라우드 세션으로.** 오래 걸리는 리서치·정리는 데스크톱 의존(Dispatch)보다 **클라우드 세션**이 안전하다 — 폰을 껐다 켜도, 지하철에서 신호가 끊겨도 서버에서 계속 [1][6].
- **결과 회수 후 '이어서 지시'.** 완성물이 오면 같은 대화에서 "이 부분만 더 다듬어"처럼 이어가면, 컨텍스트가 유지된 채 반복 개선된다 [국내 digital_snj].
- **알림 확인 루틴.** 예약·장시간 작업은 결과가 언제 오는지 정해두고(예: 점심 후 확인) 폰으로 체크하는 리듬을 만들면 "맡기고 잊기"가 실제로 굴러간다.

## 9. 요금·플랜

코워크 자체는 **유료 클로드 플랜에 포함**된 기능이다. 플랜별 차이는 **기능이 아니라 사용 용량**이다 [국내 missionary3][7]:

| 플랜 | 월 요금 | 사용 용량 | 특징 |
|---|---|---|---|
| **Pro** | 기준 | **1×** | 코워크·예약 작업 사용 가능 |
| **Max 5×** | $100 | **5×** | 헤비 작업(대량 파일 등) |
| **Max 20×** | $200 | **20×** | 기업·전문가용, 우선 접근·신기능 조기 접근 |

- **Pro와 Max는 기능이 동일**하고 **용량만 다르다**. Max는 피크 시간 **우선 접근 + 신기능 조기 접근** 포함.
- **예약 작업**은 유료 전체, **클라우드(원격) 세션**은 **베타로 Pro·Max**.
- 웹·모바일 롤아웃은 **Max 우선(2026-07-07)** 후 확대 [8][7].

## 10. Chat vs Cowork vs Claude Code

| 구분 | Chat(claude.ai) | **Cowork** | Claude Code |
|---|---|---|---|
| 성격 | 대화·조언 | **업무 실행 에이전트** | 코딩 실행 에이전트 |
| 다루는 것 | 텍스트 | 파일·폴더·앱·커넥터 | 코드·저장소·터미널 |
| 대상 | 누구나 | **비개발 지식노동자** | 개발자 |
| 표면 | 웹·데스크톱·모바일 | 데스크톱 + **웹·모바일(베타)** | 터미널·IDE·웹 |
| 자동화 | 제한적 | **예약 작업·플러그인** | 훅·스크립트 |

한마디로 **코워크 = "비개발자를 위한 Claude Code"**. 터미널 없이, 문서·파일·앱 업무를 에이전트에게 위임한다 [국내 imh1050][국내 hfeel].

## 11. 한계·주의점

베타 기능이므로 냉정하게 짚는다 [국내 imh1050][국내 6612ty][6]:
- **롤아웃 순차.** 모바일 Cowork이 아직 안 보일 수 있다(Max 우선). 앱 업데이트 후에도 없으면 대기.
- **Dispatch면 데스크톱을 켜둬야.** 로컬 실행은 컴퓨터가 자거나 앱이 닫히면 안 돈다. 완전 원격을 원하면 **클라우드 세션(베타 Pro·Max)** 사용.
- **권한·보안.** 로컬 파일·커넥터(메일·메신저·결제)에 접근하므로, **폴더 권한을 최소화**하고 민감 데이터 접근 범위를 관리해야 한다. 자동화가 편한 만큼 사고 반경도 넓다.
- **결과 검수 필수.** 에이전트가 파일을 만들고 정리하는 만큼, **덮어쓰기·삭제·오분류** 가능성에 대비해 중요한 폴더는 백업하고 결과를 확인할 것.
- **비용은 용량제.** 헤비 작업·예약 작업을 많이 돌리면 플랜 용량을 빨리 소모한다. Max 단계 선택은 실제 사용량 보고 결정.

## 12. 베스트 프랙티스 & 체크리스트

- [ ] **모바일은 '위임·확인' 용도로.** 무거운 설정보다 "결과 설명 → 회수"에 집중.
- [ ] **완전 원격이 필요하면 클라우드 세션**, 로컬 파일 필수면 **Dispatch + 데스크톱 켜두기**.
- [ ] **반복 업무는 예약 작업**으로 자동화하고 폰으로 결과만 확인.
- [ ] **폴더 권한 최소화**, 민감 커넥터(메일·결제) 접근은 신중히.
- [ ] **글로벌 지시로 말투·형식·금지사항**을 고정, 폴더별 지시로 예외 처리.
- [ ] **중요 폴더 백업 후** 정리·자동화 작업 실행. 결과는 사람이 검수.
- [ ] **앱 최신화.** Cowork 옵션이 없으면 업데이트·롤아웃 상태부터 확인.

## 13. 한눈 요약
- 코워크 = **채팅이 아니라 실행.** 대화창에서 **Cowork 켜면** 파일·앱·커넥터를 다뤄 결과물 완성.
- **모바일 확장(베타, 07-07 Max 우선)** — **어디서든 작업 배정**, 데스크톱 라우팅 또는 클라우드 세션으로 처리.
- **Dispatch(로컬, 데스크톱 켜둬야) vs 클라우드 세션(원격, 컴퓨터 꺼도 됨)** 구분이 모바일의 열쇠.
- **플러그인(스킬·커넥터·서브에이전트)** + **예약 작업(유료 전체)** 이 자동화의 축.
- **Pro/Max 기능 동일·용량만 차이**(Max 5× $100 / 20× $200). 권한 최소화·결과 검수 필수.
