# 행렬과 다변수 미적분 (입문) — 쉬운 첫걸음

> 📚 **눈높이**: 미적분을 막 시작한 중·고등학생. 기호보다 **그림과 작은 숫자**로 천천히. 새 용어는 나올 때 바로 풀어 씁니다. (이 글은 [[행렬과 다변수 미적분 개념지도]] 시리즈의 🟢입문 편)

## TL;DR
- **행렬**은 숫자를 직사각형으로 적은 '숫자 표'예요. 가로줄=행, 세로줄=열, 칸=성분 [1].
- 덧셈은 같은 자리끼리, 곱셈은 '앞의 행 × 뒤의 열'이라는 특별한 규칙을 써요(그리고 `AB≠BA`) [2][7].
- **2×2 행렬식** `ad−bc`는 "이 행렬을 되돌릴 수 있나"를 알려주는 한 숫자예요 [4][3].
- **다변수 함수**는 입력이 둘 이상(예: 산 높이 `z=f(x,y)`), **편미분**은 "한 변수만 바꾸고 나머지는 상수로 두고" 미분하는 거예요 [5][9].

## 핵심 포인트
1. 행렬 = 숫자 표. 크기는 "행×열"로 말해요(예: 2×3) [1].
2. 덧셈·실수배는 쉬워요(자리만 맞추면 끝) [1].
3. 곱셈은 '행×열 짝짓기', 순서가 중요(`AB≠BA`) [2][7].
4. 행렬식 0이면 못 되돌림(역행렬 없음) [4][3].
5. 편미분 = 하나만 움직여 경사 재기 [5].

## 본문

### 1. 행렬은 '숫자 표'예요
학생 2명, 과목 3개 점수를 표로 적어 볼게요:
```
        국어  영어  수학
 철수 [  90    80    70 ]
 영희 [  60    100   85 ]
```
이렇게 숫자를 **직사각형으로 나란히** 적은 것이 **행렬**이에요 [1].
- **행(row)**: 가로줄(철수 줄, 영희 줄)
- **열(column)**: 세로줄(국어 줄, 영어 줄…)
- **성분(원소)**: 칸 하나하나의 숫자(영희의 수학 85)
- **크기**: "행 개수 × 열 개수". 위는 행 2·열 3이라 **2×3 행렬** [1].

> 💡 **왜 표로 묶나요?** 숫자를 따로따로 다루는 대신 한 덩어리로 묶으면 **여러 계산을 한꺼번에** 할 수 있어요. 게임 그래픽, 통계, 인공지능이 다 이 '숫자 덩어리 계산'으로 돌아가요.

### 2. 덧셈과 실수배 — 가장 쉬운 부분
크기가 **같은** 두 행렬은 **같은 자리 숫자끼리** 더해요 [1]:
```
 [ 1  2 ]   [ 5  6 ]   [ 1+5   2+6 ]   [ 6   8 ]
 [ 3  4 ] + [ 7  8 ] = [ 3+7   4+8 ] = [ 10  12]
```
행렬에 그냥 숫자를 곱하는 **실수배**는 모든 칸에 똑같이 곱해요:
```
        [ 1  2 ]   [ 3   6 ]
    3 × [ 3  4 ] = [ 9  12 ]
```
여기까진 "자리만 맞으면 그대로"라고 생각하면 돼요.

### 3. 곱셈 — 규칙이 조금 특별해요
행렬 곱셈은 칸끼리 곱하는 게 **아니에요**. **앞 행렬의 한 '행'과 뒤 행렬의 한 '열'을 곱해서 더해요** [2][7].
- **조건**: 앞 행렬의 **열 개수** = 뒤 행렬의 **행 개수**. 안 맞으면 곱셈 불가 [2][8].

예제로 차근차근:
```
 [ 1  2 ]   [ 5  6 ]
 [ 3  4 ] × [ 7  8 ] = ?
```
- 결과 (1행,1열): 앞 **1행** `(1,2)` × 뒤 **1열** `(5,7)` → `1×5 + 2×7 = 19`
- 결과 (1행,2열): 앞 **1행** `(1,2)` × 뒤 **2열** `(6,8)` → `1×6 + 2×8 = 22`
- 결과 (2행,1열): 앞 **2행** `(3,4)` × 뒤 **1열** `(5,7)` → `3×5 + 4×7 = 43`
- 결과 (2행,2열): 앞 **2행** `(3,4)` × 뒤 **2열** `(6,8)` → `3×6 + 4×8 = 50`
```
 결과 = [ 19  22 ]
        [ 43  50 ]
```

> ⚠️ **순서가 중요!** 보통 `A×B`와 `B×A`는 **결과가 달라요**(`AB ≠ BA`) [7]. 숫자 곱셈(3×5=5×3)과 다른 점이에요. 직접 위 예제의 순서를 바꿔 계산해 보면 다른 값이 나와요.

### 4. 단위행렬과 역행렬 — 숫자의 '1'과 '나눗셈'
- **단위행렬 E**: 대각선만 1, 나머지는 0. 어떤 행렬에 곱해도 그대로라, 숫자의 **1** 같은 존재예요.
```
 E = [ 1  0 ]
     [ 0  1 ]
```
- **역행렬 A⁻¹**: `A × A⁻¹ = E`가 되게 하는 행렬 [3]. 숫자로 치면 '역수(나눗셈)'예요. 단, **모든 행렬이 역행렬을 갖는 건 아니에요**(있으면 딱 하나) [3].

### 5. 행렬식 — '되돌릴 수 있나' 알려주는 한 숫자
2×2에서:
```
       [ a  b ]
   A = [ c  d ]   →   행렬식 det A = a×d − b×c
```
[4]. 예: `[[1,2],[3,4]]`이면 `1×4 − 2×3 = 4 − 6 = −2`.
**행렬식이 0이 아니면 역행렬 존재(되돌릴 수 있음), 0이면 없음** [4][3]. 그래서 행렬식은 "이 행렬을 뒤집을 수 있나?"의 신호등이에요.

> 💡 **한 걸음 더(맛보기)**: 2×2 행렬을 '평면을 늘이고 비트는 변환'으로 보면, **행렬식의 크기 = 넓이가 변하는 배율**이에요. 이 아이디어가 [[야코비안 변환 입문]]으로 이어져요.

### 6. 다변수 함수 — 변수가 여러 개
지금까지의 함수는 보통 `y=f(x)`(입력 1 → 출력 1)였죠. 그런데 세상일은 여러 가지에 동시에 영향받아요.
- 산의 **높이**는 위치 `(x, y)` 두 개로 정해져요 → `z = f(x, y)`.
- `z=f(x,y)`의 그래프는 평면 위의 **언덕(곡면)**처럼 생겼어요.

### 7. 편미분 — "하나만 움직여 보기"
다변수 함수는 **방향이 여러 개**라 기울기를 하나로 못 말해요(동쪽 갈 때와 북쪽 갈 때 경사가 다름). 그래서 **편미분**:
> 관심 있는 변수 **하나만** 변화시키고, 나머지는 잠깐 **상수**로 두고 미분 [5][6][9].

기호는 평소의 `d` 대신 동글한 **`∂`**를 써요. `∂z/∂x`는 "`y`는 고정, `x`만 움직였을 때 `z`의 변화율".

**예제 1**: `f(x,y) = x² + 3y`
- `x`로 편미분(`3y`는 상수) → `∂f/∂x = 2x`
- `y`로 편미분(`x²`는 상수) → `∂f/∂y = 3`

**예제 2**: `f(x,y) = x²y`
- `x`로 편미분(`y`는 상수처럼) → `∂f/∂x = 2xy`
- `y`로 편미분(`x²`는 상수처럼) → `∂f/∂y = x²`

> 💡 **그림으로**: 언덕 위 한 점에서 **동쪽으로 자른 단면의 경사**가 `∂f/∂x`, **북쪽으로 자른 단면의 경사**가 `∂f/∂y`예요 [9].
