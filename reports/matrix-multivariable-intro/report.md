# 행렬과 다변수 미적분 — 고등학생을 위한 쉬운 입문

> 📚 **이 글의 눈높이**: 미적분을 막 시작한 고등학생. 어려운 기호보다 **그림과 말**로 먼저 이해하고, 작은 숫자 예제로 손에 익히는 걸 목표로 합니다. 한 단락에 한 개념씩, 새 용어는 나올 때 바로 풀어 씁니다.

## TL;DR
- **행렬**은 숫자를 직사각형 모양으로 나란히 적어 놓은 '숫자 표'예요. 가로줄을 **행(行, row)**, 세로줄을 **열(列, column)**이라 부르고, 칸 하나하나의 숫자를 **성분(원소)**이라 해요 [1].
- 행렬은 그냥 표가 아니라, **여러 수를 한 덩어리로 묶어 한꺼번에 계산**하게 해 줘요. 덧셈은 같은 자리끼리 더하면 되고, 곱셈은 규칙이 조금 특별해요(앞 행렬의 '열 수'와 뒤 행렬의 '행 수'가 같아야 함) [1][2][8].
- **다변수 함수**는 입력이 두 개 이상인 함수예요. 예: 산의 높이 `z = f(x, y)`는 위치 `(x, y)` 두 개를 넣으면 높이 하나가 나와요.
- 이런 함수의 기울기를 잴 때 쓰는 게 **편미분**이에요. "**한 변수만 살짝 바꾸고 나머지는 잠깐 상수로 고정**"하고 미분하는 거예요 [5][9].
- 행렬과 다변수 미적분은 결국 한 곳에서 만나요: **다변수 함수의 '미분'은 숫자 하나가 아니라 편미분들을 모아 만든 행렬**이 됩니다 — 이게 다음 단계인 [[야코비안 변환]]으로 이어져요.

## 핵심 포인트
1. **행렬 = 숫자 표**: 행(가로)·열(세로)·성분(칸). 크기는 "행×열"로 말해요(예: 2×3) [1].
2. **행렬 덧셈은 쉽다**: 같은 크기끼리, 같은 자리 숫자끼리 더해요 [1][8].
3. **행렬 곱셈은 '행 × 열' 짝짓기**: 앞 행렬의 한 '행'과 뒤 행렬의 한 '열'을 곱해 더해요. 그리고 `AB ≠ BA`(순서 바뀌면 결과도 달라짐) [2][7].
4. **편미분 = 하나만 움직이기**: 다변수 함수에서 한 변수만 변화시키고 나머지는 상수 취급해 미분 [5][6].
5. **둘의 만남**: 다변수 함수의 미분은 편미분들을 칸칸이 채운 **행렬**로 표현돼요(기울기 벡터·야코비 행렬).

## 본문

## 1부. 행렬(matrix) 이야기

### 1-1. 행렬은 '숫자 표'예요
시험 점수를 정리한다고 해 봐요. 학생 2명, 과목 3개(국어·영어·수학)라면 이렇게 표로 적을 수 있어요:

```
        국어  영어  수학
 철수 [  90    80    70 ]
 영희 [  60    100   85 ]
```

이렇게 숫자를 **직사각형 모양으로 나란히 적은 것**이 바로 행렬이에요 [1]. 용어만 익혀 둬요:
- **행(row)**: 가로줄. 위 표에서 "철수 줄", "영희 줄" 각각이 한 행이에요.
- **열(column)**: 세로줄. "국어 줄", "영어 줄"처럼요.
- **성분(원소, entry)**: 칸 하나하나의 숫자. 예를 들어 영희의 수학 점수 85가 한 성분이에요 [1].
- **크기**: "행의 개수 × 열의 개수"로 말해요. 위 행렬은 행이 2개, 열이 3개라 **2×3 행렬**이에요 [1].

> 💡 **왜 표로 묶을까?** 숫자를 하나하나 따로 다루는 대신 **한 덩어리로 묶으면, 여러 계산을 한 번에** 할 수 있어요. 컴퓨터 그래픽, 통계, 인공지능이 전부 이 '숫자 덩어리 계산'으로 돌아가요.

### 1-2. 행렬의 덧셈과 실수배 — 가장 쉬운 부분
크기가 **같은** 두 행렬은, **같은 자리 숫자끼리 더하면** 끝이에요 [1][8].

```
 [ 1  2 ]   [ 5  6 ]   [ 1+5   2+6 ]   [ 6   8 ]
 [ 3  4 ] + [ 7  8 ] = [ 3+7   4+8 ] = [ 10  12]
```

행렬에 그냥 숫자(실수)를 곱하는 **실수배**도 쉬워요 — 모든 칸에 똑같이 곱해요:

```
        [ 1  2 ]   [ 2  4 ]
    2 × [ 3  4 ] = [ 6  8 ]
```

여기까지는 "자리만 맞으면 그대로 더하고 곱한다"라고 외우면 돼요.

### 1-3. 행렬의 곱셈 — 규칙이 조금 특별해요
행렬 곱셈은 덧셈처럼 칸끼리 곱하는 게 **아니에요**. 대신 **앞 행렬의 '한 행'과 뒤 행렬의 '한 열'을 짝지어, 곱해서 더해요** [2][7].

먼저 **조건**: 앞 행렬의 **열 개수**와 뒤 행렬의 **행 개수**가 같아야 곱할 수 있어요 [2][8]. (안 맞으면 곱셈 자체가 불가능)

작은 예로 볼게요:

```
 [ 1  2 ]   [ 5  6 ]
 [ 3  4 ] × [ 7  8 ]
```

결과의 (1행, 1열) 칸은 → 앞의 **1행** `(1, 2)`와 뒤의 **1열** `(5, 7)`을 곱해 더해요: `1×5 + 2×7 = 5 + 14 = 19`.
같은 방식으로 모두 채우면:

```
 [ 1×5+2×7   1×6+2×8 ]   [ 19  22 ]
 [ 3×5+4×7   3×6+4×8 ] = [ 43  50 ]
```

> 💡 **직관**: 한 행과 한 열을 "곱해서 더하는" 이 동작은, 두 줄의 숫자를 짝지어 합치는 것과 같아요 [7]. 컴퓨터 게임에서 캐릭터를 회전·확대할 때도 이 곱셈이 쓰여요.

> ⚠️ **꼭 기억할 점: 순서가 중요해요!** 보통 `A×B`와 `B×A`는 **결과가 달라요**(`AB ≠ BA`) [7]. 숫자 곱셈(3×5=5×3)과 다른 점이에요.

### 1-4. 단위행렬과 역행렬 — 숫자의 1과 나눗셈에 해당
- **단위행렬(E)**: 대각선만 1이고 나머지는 0인 정사각 행렬. 숫자의 **1** 같은 역할이에요(어떤 행렬에 곱해도 그대로) .
- **역행렬(A⁻¹)**: 어떤 정사각 행렬 `A`에 곱해서 단위행렬이 되게 하는 행렬, 즉 `A × A⁻¹ = E` [3]. 숫자로 치면 '나눗셈(역수)'에 해당해요. 단, **모든 행렬이 역행렬을 갖는 건 아니에요**(있으면 딱 하나로 정해져요) [3].

### 1-5. 행렬식 — '뒤집을 수 있는지' 알려주는 한 숫자
**행렬식(determinant)**은 **정사각 행렬에 숫자 하나를 대응**시키는 거예요 [4]. 2×2 행렬은 아주 간단해요:

```
       [ a  b ]
   A = [ c  d ]   →   행렬식 det A = a×d − b×c
```

이 숫자가 왜 중요하냐면, **행렬식이 0이 아니면 역행렬이 존재**하고(되돌릴 수 있고), **0이면 역행렬이 없어요**(되돌릴 수 없음) [4]. 즉 행렬식은 "이 행렬을 뒤집을 수 있나?"를 알려주는 신호등이에요.

> 💡 한 걸음 더(맛보기): 2×2 행렬을 '평면을 늘이고 비트는 변환'으로 보면, **행렬식의 크기 = 넓이가 변하는 배율**이에요. 이 아이디어가 뒤에서 [[야코비안 변환]]으로 이어져요.

## 2부. 다변수 미적분 이야기

### 2-1. 변수가 여러 개인 함수
지금까지 배운 함수는 보통 `y = f(x)`처럼 **입력 1개 → 출력 1개**였죠. 그런데 세상일은 보통 여러 가지에 동시에 영향을 받아요.
- 예: 어떤 지점의 **산의 높이**는 위도와 경도, 즉 위치 `(x, y)` 두 개로 정해져요 → `z = f(x, y)`.
- 예: 난방비는 바깥 온도, 집의 단열 상태 등 **여러 변수**에 따라 달라져요 [9].

이렇게 **입력이 2개 이상인 함수**를 다변수 함수라고 해요. `z = f(x, y)`는 그래프로 그리면 평면 위의 **언덕(곡면)**처럼 생겼어요.

### 2-2. 편미분 — "하나만 움직여 보기"
1변수 함수에서 미분은 "그래프의 기울기(순간 변화율)"였어요. 그런데 다변수 함수는 **방향이 여러 개**라 기울기를 한 개로 말할 수 없어요. 동쪽으로 갈 때와 북쪽으로 갈 때 경사가 다르니까요.

그래서 나온 아이디어가 **편미분(partial derivative)**이에요. 정의는 한 문장이면 충분해요:

> **편미분 = 관심 있는 변수 하나만 변화시키고, 나머지 변수는 잠깐 '상수'로 고정한 채 미분하는 것** [5][6][10].

기호는 평소의 `d` 대신 동글동글한 **`∂`**(라운드, round)를 써요. `∂z/∂x`는 "`y`는 고정하고 `x`만 움직였을 때 `z`가 변하는 비율"이라는 뜻이에요 [9][11].

**작은 예제**: `f(x, y) = x² + 3y`
- `x`로 편미분(`y`는 상수로 봄): `3y`는 상수라 사라지고 → `∂f/∂x = 2x`
- `y`로 편미분(`x`는 상수로 봄): `x²`는 상수라 사라지고 → `∂f/∂y = 3`

> 💡 **그림으로 보면**: 곡면(언덕) 위 한 점에서 **동쪽으로 자른 단면의 경사**가 `∂f/∂x`, **북쪽으로 자른 단면의 경사**가 `∂f/∂y`예요 [9]. 즉 편미분은 "그 방향으로 갈 때 얼마나 가파른가"를 알려줘요.

### 2-3. 편미분을 모으면 '방향 정보'가 된다
한 점에서 `∂f/∂x`와 `∂f/∂y`를 **나란히 묶으면**, "어느 쪽으로 가면 가장 가파르게 올라가는지"를 알려주는 **기울기 벡터(gradient)**가 돼요. 등산 앱이 "여기서 가장 가파른 오르막 방향"을 알려주는 것과 같은 원리예요.

여기서 핵심 연결고리가 나와요: **여러 개의 편미분을 칸칸이 채워 표로 묶은 것**, 그게 바로 1부에서 배운 **행렬**이에요.

## 3부. 행렬과 다변수 미적분이 만나는 곳

입력도 여러 개, 출력도 여러 개인 함수(예: `(x, y)`를 넣으면 `(u, v)`가 나오는 변환)를 미분하면 어떻게 될까요? 출력마다·입력마다 편미분이 생기니까, **편미분이 여러 개** 나와요. 이걸 깔끔하게 담는 그릇이 바로 **행렬**이에요.

- 1변수: 미분 = 숫자 하나(기울기).
- 다변수: 미분 = **편미분들을 모은 행렬**.

이 "편미분으로 채운 행렬"이 다음 단계 주제인 **[[야코비안 변환]]**(야코비 행렬)이에요. 즉 오늘 배운
- (1부) **행렬**이라는 '그릇'과
- (2부) **편미분**이라는 '재료'가
합쳐져서 다변수 세계의 미분을 이루는 거예요. 행렬과 다변수 미적분을 따로 배우는 게 아니라, **둘이 한 팀**이라는 걸 기억하면 앞으로의 공부가 훨씬 자연스러워요.
