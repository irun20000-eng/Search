# 업무 생산성 크롬 확장프로그램 완전 활용 가이드 — 유형별 활용예시·추천법

> 작성·조회 기준일 **2026-07-12 (KST)**. 확장프로그램은 정책(Manifest V3)·기능·요금이 자주 바뀐다. 설치 전 **Chrome 웹스토어에서 사용자 수·최근 업데이트·권한**을 반드시 확인할 것.

## TL;DR
- 크롬 확장프로그램은 **브라우저 안에서 반복 작업을 없애는 작은 도구**다. 잘 고른 3~5개만으로 **업무 효율이 눈에 띄게 오른다** [1(국내)][6][7].
- 이 가이드는 **7개 유형**으로 나눠 정리한다: ①탭·세션 관리 ②글쓰기·번역 ③화면 캡처·녹화 ④노트·정보수집 ⑤작업·시간관리 ⑥AI 사이드바 ⑦보안·집중(광고차단·비밀번호) [6][7][12].
- 유형별 대표: **OneTab/Toby**(탭), **Grammarly/DeepL**(글), **GoFullPage/Loom**(캡처·녹화), **Notion Web Clipper/Raindrop**(수집), **Todoist/Clockify**(작업·시간), **Monica·Simple Chat Hub**(AI), **Bitwarden/uBlock Origin**(보안·집중) [6][7][국내].
- **핵심 원칙: "다 깔지 말고 병목부터".** 확장은 하나하나가 메모리·개인정보 노출·보안 위험을 늘린다. **3~5개 이하**로 시작하라 [6][7].
- **보안이 생명**: 무명·리뷰 적음·과도한 권한('모든 사이트의 모든 데이터')·업데이트 중단 확장은 피한다. 2026년 **Manifest V3** 전환으로 옛 광고차단 등 일부 확장이 대체됐다 [7][12].

## 핵심 포인트
1. **유형(=병목)에서 출발.** "내가 매일 어디서 시간을 버리나"를 먼저 정하고 그 유형만 [1(국내)][6].
2. **조합이 힘.** 번역+캡처+녹화를 엮으면 "외국어 자료→설명 영상"이 한 흐름이 된다 [1(국내)].
3. **3~5개 룰.** 많이 깔수록 브라우저가 느려지고 공격면이 는다 [6][7].
4. **AI 사이드바가 2026년 대세.** 페이지 맥락을 읽고 요약·번역·답변을 그 자리에서 [6(Monica)][국내 AI].
5. **보안 필수 점검.** 사용자 수·업데이트·권한 3종을 설치 전 확인 [7].

## 0. 시작 전 — 설치·관리·권한 기본기

- **설치**: Chrome 웹스토어 접속 → 검색 → "Chrome에 추가". 우측 상단 퍼즐 아이콘(📌)으로 **자주 쓰는 것만 고정** [국내 nakangdoo].
- **관리·제거**: 우측 상단 → 확장프로그램 → 관리 → 제거. 안 쓰는 건 **비활성화**만 해도 메모리를 아낀다 [국내 kong2store].
- **권한 확인(중요)**: 설치 시 "이 확장프로그램이 요청하는 권한"을 본다. **"모든 사이트의 데이터 읽기·변경"**은 강력하니, 그 확장이 정말 그럴 필요가 있는지 따진다 [7].
- **2026 정책(Manifest V3)**: 크롬이 확장 규격을 MV3로 전환하면서 일부 구형 확장이 동작을 멈추거나 대체됐다(예: uBlock Origin → **uBlock Origin Lite**). 설치 전 **MV3 지원·최근 업데이트**를 확인하라 [7][12].

### 왜 "유형"으로 접근하나

확장프로그램은 웹스토어에만 수십만 개가 있다. "TOP 10" 목록을 아무리 봐도, 정작 **내가 뭘 깔아야 하는지**는 안 나온다. 그래서 이 가이드는 도구 이름이 아니라 **"업무의 어느 지점에서 시간이 새는가"**라는 병목의 유형으로 접근한다. 사람마다 병목이 다르다 — 어떤 이는 탭이 쌓여 무너지고, 어떤 이는 영문 메일에 30분을 쓰고, 어떤 이는 같은 설명을 열 번 반복한다. **자기 병목을 알면 도구는 자동으로 정해진다.** 아래 7개 유형을 훑으며 "아, 이게 내 문제다" 싶은 곳을 두세 개만 고르면 된다. 나머지는 지금 당장 필요 없다 — 필요해지면 그때 추가하면 된다 [6][7].

## 1. 유형 ① 탭·세션 관리 — "탭 50개 지옥" 탈출

브라우저에서 가장 흔한 병목. 탭이 쌓이면 메모리도, 정신도 무너진다 [6][국내 upup11].

| 도구 | 무엇을 하나 | 활용예시 |
|---|---|---|
| **OneTab** | 열린 탭 전부를 **목록 하나로** 접기 | 리서치 후 20개 탭을 한 번에 접어 메모리 회수(최대 95% RAM 절약) [6] |
| **Toby / Workona** | 탭을 **프로젝트별 보드**로 저장·복원 | "A고객 작업" 탭 세트를 저장했다가 통째로 복원 |
| **Raindrop.io** | 북마크를 시각적으로 관리 | 자료를 태그·컬렉션으로 정리해 팀과 공유 [6] |

**활용법**: 리서치가 끝나면 **OneTab으로 접어 두고**, 반복되는 작업 묶음은 **Toby 보드**로 저장한다. "아침에 어제 하던 탭을 다시 못 찾는" 문제가 사라진다 [6]. 특히 노트북 메모리가 부족해 브라우저가 버벅인다면, 탭 관리 확장 하나만으로도 **체감 속도가 가장 극적으로** 바뀌는 유형이 이것이다 [6][국내 upup11].

## 2. 유형 ② 글쓰기·번역 — 문서·메일 품질 자동 상향

브라우저에서 글을 많이 쓰는 사람(메일·문서·블로그)에게 즉효 [7][국내 ha-term].

| 도구 | 무엇을 하나 | 활용예시 |
|---|---|---|
| **Grammarly** | 영문 문법·톤·명료성 교정, 문장 재작성·AI 초안 | 영문 메일을 쓰면 실시간으로 오류·어색한 표현 교정 [7] |
| **DeepL / Papago** | 고품질 번역(선택 영역·페이지) | 외국어 자료를 드래그해 즉시 번역, 답장 초안 작성 [국내 sahas] |
| **Wordtune** | 문장 바꿔쓰기·톤 조절 | 딱딱한 문장을 부드럽게/격식 있게 |

**활용법(조합)**: 외국어 문의가 오면 **DeepL로 이해 → Grammarly로 답장 교정** → 몇 분 만에 매끄러운 영문 회신. 국내 가이드도 "외국어 업무 대응"을 대표 조합 시나리오로 든다 [국내 sahas].

## 3. 유형 ③ 화면 캡처·녹화 — 말 대신 보여주기

"글로 설명하면 열 줄, 영상으로 보여주면 30초"인 상황에 강력하다 [6][국내 upup11].

| 도구 | 무엇을 하나 | 활용예시 |
|---|---|---|
| **GoFullPage** | **전체 페이지 통 캡처**(스크롤 포함) | 긴 웹페이지·대시보드를 한 장 이미지로 보고서에 첨부 [국내 upup11] |
| **Loom** | 화면·카메라·음성 **원클릭 녹화 + 링크 공유** | 버그 재현·기능 시연·피드백을 영상으로 비동기 전달 [6][7] |
| **Awesome Screenshot** | 영역 캡처·주석·간단 편집 | 스크린샷에 화살표·설명 달아 즉시 공유 |

**활용법**: 회의를 잡는 대신 **Loom 3분 녹화**로 설명을 대체한다. 2026년 Loom은 **AI가 자동으로 자막·요약·할 일**을 뽑아줘, 받는 사람이 영상을 다 안 봐도 핵심을 안다 [6][7].

## 4. 유형 ④ 노트·정보수집 — 흩어진 자료를 한곳으로

리서치·기획 업무의 핵심. "본 것을 남기고 다시 찾는" 문제를 푼다 [7].

| 도구 | 무엇을 하나 | 활용예시 |
|---|---|---|
| **Notion Web Clipper** | 웹페이지를 **노션 DB로** 저장(전문·선택·링크) | 참고 기사·경쟁사 페이지를 프로젝트 DB에 자동 적재 [7] |
| **Raindrop.io** | 북마크 시각 관리·태그·팀 공유 | 자료를 주제별 컬렉션으로 [6] |
| **Evernote/Obsidian 클리퍼** | 개인 지식베이스로 스크랩 | 나만의 볼트에 하이라이트 저장 |

**활용법**: 리서치 중 좋은 자료를 만나면 **Notion Web Clipper로 즉시 클립** → 나중에 노션에서 검색·정리. 앞서 다룬 "단일 정본에 모으기"의 브라우저판이다 [7].

## 5. 유형 ⑤ 작업·시간관리 — 할 일과 시간을 눈에 보이게

무엇을·언제 할지, 시간을 어디에 썼는지 관리 [7][국내 sa03134].

| 도구 | 무엇을 하나 | 활용예시 |
|---|---|---|
| **Todoist** | 어떤 웹페이지에서든 **할 일 추가**, 자연어 날짜 | 메일을 할 일로 전환, 아침에 AI가 급한 것 우선순위 제시 [7] |
| **Clockify** | 작업별 **시간 추적**(무료) | 프로젝트별 투입 시간 측정→청구·회고 [7] |
| **Forest / BlockSite** | 집중 타이머·사이트 차단 | 업무 시간에 SNS 차단, 뽀모도로 집중 [6] |

**활용법**: 재택·프리랜서라면 **Clockify로 시간 측정 → 어디서 새는지 확인**하고, **Todoist로 할 일 원클릭 캡처**. 국내에도 "재택 생산성 2배" 시간추적 활용기가 많다 [국내 sa03134].

## 6. 유형 ⑥ AI 사이드바 — 2026년의 게임체인저

페이지를 **떠나지 않고** 요약·번역·답변·초안을 받는 AI 도우미. 2026년 생산성 확장의 최대 트렌드다 [6][국내 mksking].

| 도구 | 무엇을 하나 | 활용예시 |
|---|---|---|
| **Monica AI** | GPT·Claude 기반 사이드바, **현재 페이지를 읽고** 요약·번역·질의응답·작성 | 긴 기사를 3줄 요약, 이 페이지 근거로 질문 답변 [6] |
| **Simple Chat Hub** | 여러 AI(챗GPT·클로드·제미나이)를 **한 창에서 동시** | 같은 프롬프트를 여러 모델에 던져 비교 [국내 mksking] |
| **웹 요약/번역 계열** | 유튜브 자막 요약·PDF 요약 | 영상·문서를 빠르게 파악 |

**활용법**: "이 페이지 읽고 핵심만" → 사이드바 AI가 **맥락을 읽어** 별도 창을 열지 않고 답한다. 별도 챗봇 창을 오가는 것보다 훨씬 빠르다는 게 핵심 이점이다 [6]. 국내에선 "여러 AI를 한 창에서 비교"하는 활용이 인기 [국내 mksking].

**깊이 있는 활용 팁**: AI 사이드바의 진짜 힘은 **"페이지 위에서 바로 일이 끝난다"**는 데 있다. 예컨대 긴 계약서 페이지에서 "독소조항 있나 짚어줘", 경쟁사 소개 페이지에서 "우리 제품과 비교표로 만들어줘", 영문 기술문서에서 "이 API 사용 예제를 파이썬으로", 유튜브 강의에서 "핵심을 타임스탬프와 함께 요약" — 이 모두를 탭 전환 없이 처리한다. 다만 주의할 점은 **민감 정보**다. 사이드바 AI는 페이지 내용을 외부 서버로 보내 처리하므로, **사내 기밀·고객 개인정보가 있는 페이지에서는 신중**해야 한다(§10 보안). 무료 요약·번역은 편리하지만, 무엇이 어디로 전송되는지 개인정보 정책을 한 번은 확인하는 게 좋다 [6][7].

## 7. 유형 ⑦ 보안·집중 — 광고차단·비밀번호·다크모드

효율의 토대다. 느리고 산만하고 위험한 브라우저는 그 자체가 생산성 손실이다 [7][국내 ha-term].

| 도구 | 무엇을 하나 | 활용예시 |
|---|---|---|
| **uBlock Origin Lite** | 광고·트래커 차단(MV3 대응) | 페이지 로딩 가속·집중 방해 제거 [7][12] |
| **Bitwarden / 1Password** | 비밀번호 생성·자동입력·동기화 | 계정마다 강한 암호, 로그인 자동화(무료는 Bitwarden) [6][7] |
| **Dark Reader** | 모든 사이트 다크모드 | 야간 작업 눈 피로↓ [7] |
| **Momentum** | 새 탭을 집중 대시보드로 | 새 탭 열 때 오늘 할 일·집중 문구 |

**활용법**: **Bitwarden 하나로** 모든 로그인을 자동화하면 "비밀번호 까먹어 헤매는" 시간이 사라진다. **uBlock Origin Lite**로 광고를 걷으면 리서치 속도가 체감된다 [6][7].

## 8. Worked 예시 — 직군별 추천 스택

**"다 깔지 말고, 내 직군의 병목 3~5개"** 원칙으로 조합한 예다 [6][7].

```
[마케터/리서처]
 OneTab(탭 정리) + Notion Web Clipper(자료수집) + DeepL(번역)
 + GoFullPage(캡처) + Monica(요약)
 → "자료 수집→요약→정리"가 한 흐름

[개발자/PM]
 Toby(프로젝트 탭) + Loom(버그 시연) + Todoist(작업)
 + Bitwarden(보안) + uBlock Origin Lite(속도)
 → "이슈 재현→전달→추적"이 매끄럽게

[영업/CS]
 Grammarly(영문 응대) + DeepL(번역) + Loom(제품 시연)
 + Clockify(응대 시간) + 1Password(계정)
 → "빠르고 정확한 고객 커뮤니케이션"

[1인 크리에이터]
 Simple Chat Hub(AI 비교) + GoFullPage(캡처)
 + Raindrop(자료) + Forest(집중) + Grammarly
 → "기획→집필→집중"
```

핵심: **각 스택은 5개 이하**다. 유형이 겹치지 않게(탭 하나, 글 하나, 캡처 하나…) 골라야 서로 방해하지 않는다 [6].

**조합의 시너지 — 한 시나리오를 끝까지.** 마케터 스택으로 "해외 경쟁사 분석 리포트"를 만든다고 하자. ①해외 자료를 리서치하며 탭이 쌓이면 **OneTab**으로 접어 정리, ②좋은 자료는 **Notion Web Clipper**로 프로젝트 DB에 클립, ③외국어 본문은 **DeepL**로 번역해 이해, ④핵심 대시보드·차트는 **GoFullPage**로 통 캡처해 리포트에 첨부, ⑤긴 기사는 **Monica**로 3줄 요약해 인용. 다섯 도구가 각자 다른 칸을 채우며 **"수집→이해→정리→인용"이 끊김 없이** 이어진다. 반대로 같은 유형을 두 개 깔면(예: 번역기 두 개) 단축키가 충돌하고 어느 걸 쓸지 헷갈릴 뿐이다. **한 유형에 하나씩**이 조합의 제1 규칙이다 [1(국내)][6].

## 9. 도입 4단계 — 어떻게 시작할까

```
1단계  병목 찾기: "매일 어디서 시간을 버리나?" 한 가지만 적는다.
2단계  그 유형에서 1개 설치(사용자 수·업데이트·권한 확인).
3단계  1~2주 써보고 정말 쓰는지 확인. 안 쓰면 제거.
4단계  다음 병목으로 이동, 총 3~5개까지만 누적.
```

- **한 번에 다 깔지 말 것.** "설치는 공짜지만 유지비(메모리·주의·보안)는 유료"다 [6][7].
- **정기 점검**: 분기마다 안 쓰는 확장 제거. 업데이트 끊긴 것은 특히 위험 [7].
- **"2주 규칙"**: 새 확장은 2주 써보고 **실제로 손이 갔는지** 돌아본다. 안 썼다면 그 병목은 도구로 풀 문제가 아니었던 것이니 미련 없이 제거한다. 이 습관만으로도 "깔아만 두고 방치된 확장"이 쌓이는 걸 막는다 [6].

## 10. 보안·주의 — 꼭 지킬 것

- **설치 전 3종 확인**: ①**사용자 수**(많을수록 검증됨) ②**최근 업데이트**(관리되는가) ③**요청 권한**(과도하지 않은가) [7].
- **위험 신호**: 무명 개발사·리뷰 소수·"모든 사이트 데이터" 권한·오랜 업데이트 중단·갑작스러운 소유권 이전. 이런 확장은 **개인정보 수집·악성코드**로 돌변할 수 있다 [7].
- **권한 최소화**: 특정 사이트에서만 필요하면 "사이트 접근"을 **클릭 시에만**으로 제한한다.
- **비밀번호·금융 관련**은 **평판 확실한 대형 확장**만(Bitwarden·1Password 등) [6][7].
- **MV3 이후**: 옛 확장이 멈추면 억지로 우회하지 말고 **MV3 대응 대체품**을 쓴다(보안·안정성) [12].
- **회사 정책 확인**: 사내 기밀을 다루면 **회사가 허용한 확장**만. 데이터가 외부 서버로 나갈 수 있다.

## 11. 흔한 오해 · FAQ

- **"많이 깔수록 좋다?"** — 반대다. 각 확장이 **메모리·속도·보안**을 갉는다. **3~5개**가 황금률 [6][7].
- **"무료면 안전?"** — 무료 확장은 종종 **데이터로 수익**을 낸다. 권한·개인정보 정책을 확인 [7].
- **"AI 사이드바가 챗봇 창보다 나은 이유?"** — **현재 페이지 맥락을 읽어** 답하기 때문. 복붙 왕복이 없다 [6].
- **"업데이트가 왜 중요?"** — 크롬 정책(MV3)·보안 패치를 따라가야 한다. **방치된 확장은 취약점** [7][12].
- **"모바일에서도 되나?"** — 크롬 확장은 기본적으로 **데스크톱** 중심이다(일부 안드로이드 브라우저 예외).
- **"확장 대신 SaaS/앱을 쓰면 안 되나?"** — 상황에 따라 낫다. 확장은 **브라우저 안에서 즉시** 쓰는 게 강점이고, 무거운 작업(대량 자동화·팀 협업)은 전용 앱/서비스가 낫다. 확장은 "브라우저에서 반복되는 작은 마찰"을 없애는 데 최적이라고 보면 된다 [7].
- **"단축키를 외워야 하나?"** — 자주 쓰는 2~3개만 단축키를 정해두면 체감 속도가 확 오른다(캡처·번역·AI 사이드바 호출 등). 확장 관리 페이지에서 키를 지정할 수 있다 [국내 nakangdoo].
