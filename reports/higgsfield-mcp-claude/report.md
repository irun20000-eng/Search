# 클로드 코드 × 힉스필드(Higgsfield) MCP — 연결·활용·설치·예제·비용 완전 가이드

> 작성·조회 기준일 **2026-07-12 (KST)**. 요금·크레딧·모델 목록은 **매우 빠르게 바뀐다**(무료화·플랜 개편 잦음). 결제·생성 전 반드시 공식(higgsfield.ai/mcp, higgsfield.ai/cli)에서 최신값을 확인할 것. 선행 개념: [[MCP(Model Context Protocol)]] 완전 가이드, 유사 도구 [[Google Flow]].

## TL;DR
- **힉스필드(Higgsfield)**는 이미지 한 장을 **영화처럼 움직이는 영상**으로 만드는 AI 플랫폼이고, **MCP 서버**를 공식 제공해 **클로드 안에서 대화로 이미지·영상을 생성**하게 해준다. 사이트를 오갈 필요가 없어진다 [1(국내)][6][7].
- 서버는 **호스티드(원격) MCP**다: 주소 **`https://mcp.higgsfield.ai/mcp`**, 인증은 **OAuth(힉스필드 계정 로그인)** — **API 키가 필요 없다** [6][8]. (CLI를 쓸 때만 별도 API 키를 발급받는다 [7].)
- Claude Code 설치는 한 줄: **`claude mcp add --transport http --scope user higgsfield https://mcp.higgsfield.ai/mcp`** → `/mcp`로 계정 인증 [8][국내 3].
- MCP로 열리는 대표 모델: **GPT Image 2·Soul V2(이미지)**, **Veo 3.1·Kling 3.0·Sora·Minimax Hailuo·Flux 2(영상/이미지)** 등 — **4K 이미지·최대 15초 영상·캐릭터 일관성(Soul)·시네마틱 카메라 무빙** [6][7].
- 비용: **무료 150 크레딧/월**(≈GPT Image 2 15장 또는 Veo 3.1 2클립), 유료는 **Starter $15 · Plus $39(연) · Ultra $99** 대. **크레딧은 모델·해상도에 따라 차감**되므로 계획적으로 [6][9][국내 1].

## 핵심 포인트
1. **"채팅으로 영상 제작".** 클로드가 기획·프롬프트를, 힉스필드가 생성을 맡는다 [1(국내)][2(국내)].
2. **API 키 불필요(호스티드).** OAuth 로그인만으로 연결 — 진입장벽이 낮다 [6][8].
3. **한 줄 설치.** `claude mcp add --transport http --scope user …` [8].
4. **크레딧 경제.** 무료는 넉넉지 않다. **모델·해상도별 소모**를 알고 아껴 써야 한다 [9][국내 1].
5. **유사 도구와 위치.** [[Google Flow]]가 "감독 스튜디오"라면, 힉스필드 MCP는 **"클로드 대화에 영상 생성을 붙이는 커넥터"**다.

## 1. 무엇인가 — 힉스필드 · MCP · 그 조합

- **힉스필드(Higgsfield)**: 정적 이미지를 **실제 카메라로 찍은 듯 움직이는 영상**으로 바꾸는 AI 플랫폼. 줌인·줌아웃·패닝·돌리·트래킹 같은 **시네마틱 카메라 무빙**과 다수의 영상 모델을 제공한다 [국내 malgnlms][국내 toresi].
- **MCP(Model Context Protocol)**: AI(클로드)가 외부 도구와 대화하는 **표준 연결 규격**(자세히는 [[MCP(Model Context Protocol)]]). 힉스필드가 **공식 MCP 서버**를 내놓으며, 클로드가 힉스필드의 생성 기능을 **직접 호출**할 수 있게 됐다 [6][국내 moonas2002].
- **조합의 의미**: 예전엔 "클로드에서 기획 → 복사 → 힉스필드 사이트에 붙여넣기 → 생성 → 다시 가져오기"를 반복했다. MCP 연결 후엔 **클로드 대화창 하나에서 기획·이미지·영상이 끝난다** [1(국내)][2(국내)]. 국내 후기 표현으로 "영상 제작이 **채팅이 됐다**" [2(국내)].

### 왜 지금 화제인가 — "복붙의 종말"

이 조합이 2026년 여름 국내외에서 갑자기 뜨거워진 이유는 단순하다. **마찰이 사라졌기 때문**이다. AI로 영상을 만들려면 원래 여러 도구를 오갔다 — 챗봇에서 아이디어를 얻고, 이미지 툴에서 컷을 만들고, 영상 툴에 넣고, 결과를 다시 편집기로 옮겼다. 각 단계마다 창을 바꾸고, 프롬프트를 복사하고, 파일을 다운로드·업로드했다. MCP는 이 **"창 사이를 오가는 복붙 노동"을 없앤다.** 클로드가 힉스필드를 도구처럼 직접 부르니, 사용자는 **한 대화에 머물며 말로 지시**하면 된다. 국내 후기들이 하나같이 "이제 영상 제작이 채팅이 됐다", "여러 사이트를 이동할 필요가 없다"고 말하는 게 이 지점이다 [2(국내)][3(국내)]. 앞서 다룬 [[Google Flow]]가 "웹 스튜디오 안에서 감독처럼 편집"하는 접근이라면, 힉스필드 MCP는 **"내가 이미 쓰는 클로드에 영상 생성 능력을 이식"**하는 접근이다 — 둘은 경쟁이라기보다 **다른 진입점**이다.

## 2. 무엇을 할 수 있나 — 노출 모델·기능

힉스필드 MCP는 여러 이미지·영상 모델을 **한 커넥터로** 묶어 노출한다(플랫폼 전체는 30종+, MCP는 대표 모델군을 제공) [6][7].

| 구분 | 대표 모델 | 무엇 |
|---|---|---|
| **이미지** | GPT Image 2, **Soul V2**, Flux 2, Seedream | 텍스트/참조 → 고품질 이미지(최대 **4K**) [6] |
| **영상** | **Veo 3.1**, **Kling 3.0**, Sora, Minimax Hailuo | 이미지/텍스트 → 영상(최대 **15초**), 시네마틱 [6][7] |
| **일관성** | Soul(캐릭터 트레이닝) | 여러 컷에서 **같은 인물** 유지 [6] |
| **카메라** | (플랫폼 프리셋) | 줌·패닝·돌리·트래킹 등 무빙 [국내 malgnlms] |

핵심: **한 창에서 "이미지 만들기 → 그 이미지를 영상으로 → 여러 모델 비교"** 를 자연어로 지시한다. Kling 3.0 **멀티샷**으로 여러 컷 시네마틱 영상을 만드는 활용이 국내 강의로도 소개된다 [국내 brandmite].

## 3. 설치 ① Claude Code (터미널) — 한 줄

가장 빠른 경로다. 터미널에서 명령 한 줄 [8][국내 3]:

```bash
claude mcp add --transport http --scope user higgsfield https://mcp.higgsfield.ai/mcp
```

- **옵션 해설**: `--transport http`(원격 HTTP 서버) · `--scope user`(내 모든 프로젝트에서 사용) · `higgsfield`(내가 붙일 이름) · 끝의 URL이 서버 주소.
- **인증**: 클로드 코드에서 **`/mcp`** 입력 → higgsfield 항목에서 **Authenticate** → 브라우저가 열리며 **힉스필드 계정 로그인(OAuth)** → 승인하면 연결 완료 [8].
- **확인**: `claude mcp list`로 연결 상태를 본다. 연결되면 "이미지/영상 만들어줘"를 바로 지시할 수 있다.
- **스코프 선택**: 이 프로젝트에서만 쓰려면 `--scope project`(팀과 `.mcp.json` 공유), 나만 쓰려면 `--scope user`. 영상 생성은 개인 계정 인증이라 보통 **user**가 편하다 [8].

## 4. 설치 ② Claude 웹 / Desktop / Cowork — 클릭 연결

터미널이 부담스러우면 GUI로도 된다 [6][국내 4]:

1. Claude(웹/Desktop/Cowork)의 **커넥터/통합 설정** 열기.
2. **Add(추가) → Connect** 에서 힉스필드 MCP를 추가(주소 `https://mcp.higgsfield.ai/mcp`).
3. **힉스필드 계정으로 로그인(OAuth)** → 승인.
4. 끝. 이제 "이 이미지를 영상으로 만들어줘"처럼 **자연어로 지시**하면 된다 [6].

> 지원 클라이언트: **Claude(웹·Cowork·Code)** 외에 OpenClaw·Hermes Agent·NemoClaw 등 MCP 호환 에이전트에서도 동일하게 붙는다 [6].

## 5. 인증·API 키 — 헷갈리기 쉬운 지점

- **호스티드 MCP 서버**(위 방식): **API 키 불필요.** 인증은 **OAuth(계정 로그인)** 뿐이다. 키를 발급·보관·설정할 필요가 없어 간편하다 [6][8].
- **힉스필드 CLI**(터미널 전용 별도 도구)를 쓰는 경우엔 **API 키가 필요**하며, **cloud.higgsfield.ai/api-keys**에서 발급한다 [7]. (MCP만 쓸 거면 신경 쓸 필요 없다.)
- **키 보안**: CLI 키를 쓸 땐 환경변수/시크릿으로 관리하고 리포에 평문 커밋 금지(일반 MCP 보안 원칙, [[MCP(Model Context Protocol)]]).

## 6. 사용법 — 자연어로 "이미지 → 영상"

연결 후엔 **프롬프트 없이도** 대화로 지시한다. 국내에서 "프롬프트 없이 AI 영상 만드는 법"으로 소개되는 이유다(클로드가 프롬프트를 알아서 구성) [국내 jduk1228].

**기본 흐름**
```
1) "○○ 컨셉의 이미지를 GPT Image 2로 만들어줘" (필요시 참조 이미지 첨부)
2) "그 이미지를 Veo 3.1로 5초 영상으로, 천천히 줌인" 
3) "같은 인물로 다른 앵글도 하나 더" (Soul 일관성)
4) "Kling 3.0으로도 만들어 비교해줘"
5) 마음에 드는 결과를 저장/다운로드
```

- 클로드가 **어떤 모델·해상도·길이가 좋을지 제안**하고, 힉스필드가 생성한다.
- **여러 모델 비교**를 한 대화에서 시킬 수 있는 게 강점(모델별 화풍·모션이 다르므로).

**왜 "프롬프트 없이"가 되나.** 일반 영상 툴은 사용자가 직접 정교한 프롬프트(카메라·조명·분위기·오디오)를 써야 좋은 결과가 나온다([[Google Flow]] 실습편의 프롬프트 공식 참고). 그런데 힉스필드 MCP에서는 **클로드가 그 프롬프트를 대신 설계**한다. 사용자가 "카페 신메뉴를 감성적으로"라고만 말해도, 클로드가 "웜톤 조명, 얕은 심도, 천천히 돌리인, 9:16 세로" 같은 구체적 지시로 번역해 힉스필드에 넘긴다. 즉 **프롬프트 엔지니어링의 부담을 클로드가 흡수**한다 — 이것이 국내에서 "프롬프트 없이 AI 영상"이라 불리는 실체다 [국내 jduk1228]. 물론 원하는 방향이 뚜렷하면 직접 구체적으로 지시하는 게 더 정확하다. 클로드에게 "카메라는 고정, 색은 차갑게"처럼 **핵심 제약만 얹어주면** 나머지를 알아서 채운다.

## 7. Worked 예제 — 30초 쇼츠 한 편 만들기

실제 콘텐츠 하나를 클로드 대화만으로 뽑는 흐름이다 [1(국내)][국내 myself_777].

```
[목표] 카페 신메뉴 홍보 세로형 쇼츠(약 30초)

1) 기획(클로드): "카페 아이스 음료 신메뉴 30초 쇼츠 콘티 4컷 짜줘"
   → 클로드가 컷별 장면·카피 제안
2) 이미지(힉스필드 MCP): "컷1: 창가 우드톤 카페의 아이스 음료 클로즈업,
   GPT Image 2, 세로 9:16, 웜톤" → 이미지 생성
3) 영상화: "이 컷을 Veo 3.1로 5초, 카메라 천천히 돌리인" → 클립
4) 반복: 컷2~4도 같은 방식(Soul로 브랜드 톤 유지)
5) 비교: "컷3은 Kling 3.0으로도 만들어줘" → 더 나은 쪽 채택
6) 마무리: 각 클립 다운로드 → 편집앱에서 이어붙이고 자막·음악
```

- **핵심**: 클로드가 **감독·기획**을, 힉스필드가 **촬영·생성**을 맡는다. 사이트 이동·복붙이 사라져 국내 후기 기준 "10일 작업이 10분"이라는 표현까지 나온다(과장 감안) [국내 nce0623].
- **크레딧 유의**: 4K·긴 영상·여러 모델 비교는 크레딧을 크게 먹는다 → §9 절약 전략.

## 9. 크레딧 절약 전략 — "태워먹지 않기"

무료·저가 플랜에서 크레딧은 금방 녹는다. 실무 절약법 [9][국내 1]:

- **저해상도·짧은 길이로 먼저 확정** → 구도·모션이 마음에 들면 그때만 고해상도/길게 재생성.
- **모델 비교는 확정 컷에만.** 모든 컷을 여러 모델로 뽑으면 크레딧이 배로 든다.
- **이미지 먼저 확정 후 영상화.** 영상 생성이 이미지보다 비싸므로, **좋은 이미지를 골라** 영상으로.
- **프롬프트를 클로드에 다듬게** 한 뒤 생성(재생성 횟수↓).
- **무료 크레딧은 실험용**, 실제 프로젝트는 목적에 맞는 유료 플랜으로.

**크레딧 감각을 몸에 익히기.** 처음 쓰는 사람이 가장 당황하는 건 "생성 몇 번 했는데 크레딧이 왜 벌써 없지?"다. 무료 150 크레딧은 넉넉해 보여도 **Veo 3.1 영상은 클립당 수십 크레딧**이 들 수 있어 몇 번이면 소진된다. 그래서 유료 결제 전에 반드시 **"내가 한 달에 영상 몇 개를 만들 것인가"**를 먼저 셈해야 한다. 예컨대 주 3개 쇼츠(월 12개)를 목표한다면, 각 쇼츠가 4~6클립 + 재생성 여유분을 감안해 필요한 크레딧을 역산하고, 그에 맞는 플랜(대개 Plus 이상)을 고른다. 국내 가이드들이 공통으로 "목적별로 플랜을 고르라"고 강조하는 이유가 이 역산에 있다 [국내 1][국내 nce0623]. 반대로 "가끔 재미로"라면 무료로 충분하니 유료 결제를 서두를 필요가 없다.

## 10. 한계·주의 — 정직하게

- **크레딧 소모**: 4K·15초·다모델 비교는 비싸다. **무료 150은 실험 수준** [6][9].
- **모델별 편차**: Veo·Kling·Sora는 화풍·모션·강점이 다르다. **한 모델이 만능이 아니다** — 용도별로 골라야 [6].
- **길이 한계**: 한 클립 **최대 15초 수준**. 긴 영상은 **여러 클립을 이어붙여** 만든다(편집 필요) [6].
- **상업 이용·저작권**: 유명인 얼굴·저작권 캐릭터 제한, 결과물 상업 사용 시 **플랜별 라이선스·워터마크 정책** 확인.
- **초기 단계**: 국내 후기도 "아직 초기라 결과 편차가 있다"고 지적 [국내 civil6060]. 중요한 상업물은 **사람 검수** 필수.
- **정책 변동성**: 무료화·요금이 급변하므로 **의존 전 비용을 시뮬레이션**하라.

## 11. 활용 시나리오 — 누구에게 유익한가

- **1인 크리에이터/쇼츠 제작자**: 기획→생성을 한 창에서. 제작 속도↑ [국내 myself_777].
- **소상공인·마케터**: 제품 홍보 영상·상세페이지 컷을 저비용으로.
- **광고 기획자**: 컨셉 시안·프리비주얼을 빠르게 뽑아 발표.
- **교육/강의**: "클로드×힉스필드" 워크플로우 자체가 인기 강의 주제 [국내 brandmite].
- **수익화 관심자**: "AI 영상 자동화→수익화" 콘텐츠가 국내에서 활발 — 단, **크레딧 비용 대비 산출**을 계산하고 시작 [국내 myself_777].

## 12. 시작 체크리스트

**DO**
- **무료 계정으로 먼저** 연결·실험(150 크레딧) [6].
- Claude Code면 **`claude mcp add … https://mcp.higgsfield.ai/mcp`** → `/mcp` 인증 [8].
- **이미지 확정 → 영상화**, 저해상도로 구도 잡고 고해상도는 마지막 [9].
- **모델 비교는 핵심 컷만**(크레딧 절약).
- 결제 전 **공식 요금·라이선스 재확인**(정책 급변) [6][9].

**DON'T**
- 무료 크레딧으로 상업 대량생산 시도하지 말 것(금방 소진) [6].
- 모든 컷을 4K·다모델로 뽑지 말 것(크레딧 폭증) [9].
- 유명인·저작권 캐릭터 생성 금지.
- 중요한 상업물을 **검수 없이** 그대로 쓰지 말 것 [국내 civil6060].
